**UI first.** This is a pure UI behavior test (frontend pagination after `router.push`). No API calls needed for validation. No DB writes by the test (the clone creates programs via UI modal, which calls `POST /uown/clonePrograms` internally).

**Triple Validation: N/A.** This task validates UI pagination behavior after a frontend redirect. There is no new API endpoint, no new DB table, and no data-display page being introduced. The only DB query is a `SELECT` in `beforeAll` to obtain a `programPk` for navigation.

## Implementation Notes

1. **Serial mode required:** CT-02 depends on CT-01 completing the clone and landing on `/programs`. Use `test.describe.configure({ mode: 'serial' })` and share state via `let` variables.

2. **Do NOT call `showMaxRows()` after redirect:** The existing `navigateToPrograms()` method calls `showMaxRows()` internally (L19). For this test, do NOT use `navigateToPrograms()` after the clone redirect. Instead, let the page load naturally and only wait for `SELECTORS.tableRow` to appear. This preserves the "natural" pagination state that the fix should enforce.

3. **New page object methods (Task #1214 section):** Add 7 new methods to `ProgramsPage`:
   - `openCloneGroupModal()` -- clicks "Clone Group" button, waits for modal
   - `fillCloneGroupName(name)` -- fills `input[name="programGroupName"]`
   - `selectAllProgramsInModal()` -- clicks `input[name="selectAll"]`
   - `submitCloneGroupModal()` -- clicks SAVE in modal
   - `getTableRowCount()` -- returns `SELECTORS.tableRow` count
   - `getRowsPerPageValue()` -- returns dropdown value
   - `isNextPageEnabled()` -- returns enabled state of next page button

4. **Modal interaction sequence:** The `CloneProgramGroup` modal loads programs asynchronously on open (L162-165). After opening the modal, wait for at least one program checkbox to appear before clicking "Select All". The `filteredPrograms.map()` at L273 renders individual checkboxes -- wait for `input[name^="p-"]` to be visible.

5. **Selector strategy for modal elements:** The modal uses `common-ui` `<Modal>` component which renders Bootstrap-style `.modal.show`. Use:
   - Modal visibility: `SELECTORS.modalShow`
   - Group name input: `input[name="programGroupName"]` (inline in page object, or add to SELECTORS)
   - Select All: `input[name="selectAll"]`
   - SAVE button: `page.locator(SELECTORS.modalShow).getByRole('button', { name: 'SAVE' })` -- scoped to modal to avoid conflict with form SAVE
   - Program checkboxes: `input[name^="p-"]`

6. **Authentication pattern:** Follow the established pattern from #1252 test: `loginToPortalWithOptions(page, testEnv.originationUrl, testEnv)`. No storageState (origination auth is commented out in auth.setup.ts per memory).
