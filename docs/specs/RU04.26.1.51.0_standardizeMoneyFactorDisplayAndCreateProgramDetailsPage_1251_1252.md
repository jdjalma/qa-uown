## Edge Cases

1. **Money Factor is NULL in DB** — `toMoneyFactorDisplayValue(null)` returns `""` (empty string). Tests must pick a program with `money_factor > 0 AND money_factor IS NOT NULL`. If no such program exists in the environment, the test should skip gracefully with a warning.

2. **Trailing zeros** — DB `0.150000` → `toMoneyFactorDisplayValue` produces `"15"` (not `"15.0000"` or `"15.0"`). The `trimTrailingZeros` regex strips `.0+$` and `[1-9]0+$` patterns. Test must use the exact same algorithm for expected value computation.

3. **Precision edge: numeric(9,6)** — The DB column `money_factor` has 6 decimal places. `0.169999` is stored exactly. `(0.169999 * 100).toFixed(4)` = `"16.9999"`. No precision loss expected.

4. **Form value vs display value** — The table uses `toMoneyFactorDisplayValue()` (string with trailing zeros trimmed); the form uses `toMoneyFactorFormValue()` (numeric `value * 100`). When comparing, the test must handle type coercion: the form input value may be a string like `"16.9999"` while the expected is also `"16.9999"`. Both functions produce the same visual result for the same input.

5. **Sort indicators in table headers** — The Programs table uses `react-data-table-component` with sortable columns. `ProgramsPage.getCleanHeaders()` already strips Unicode sort indicators (`[triangles/arrows]`). No additional handling needed.

6. **Pagination** — The program may not be on the first page. `ProgramsPage.findProgramRow()` already handles pagination (iterates pages). No additional handling needed.

7. **Permission-gated buttons** — Clone Program Group button is only visible if the user has `manage_program_groups` permission. CT-09 should use a soft assertion for this button (log and continue if not visible).

8. **Activity Log panel conditional render** — `ActivityLogPanel` is only rendered when `programsFormik.values.programPk` is truthy (source: `[programPk].tsx:213`). For new programs (`/programs/new`), it will not appear. CT-09 tests an existing program, so it should be visible.

9. **Save toast vs redirect race** — After `showToast('success', 'Program saved successfully')`, the code immediately calls `router.push('/programs')`. The toast may disappear quickly due to page navigation. Use `Promise.all` or capture toast before redirect assertion.

10. **CT-10 cleanup** — Editing a program's `amountChargedAtSigning` is a write operation. The test must restore the original value or use a value that is identical (effectively a no-op save). Recommendation: read the original value, write the same value (testing save mechanics without data mutation), or use a known test-only program.

## toMoneyFactorDisplayValue — Reference Implementation (for test assertions)

```typescript
/**
 * Replicates ../origination/utils/money-factor.ts logic for test assertions.
 * Use this to compute expected display values from raw DB money_factor.
 */
function toMoneyFactorDisplayValue(value: number | string | null | undefined): string {
  if (value === null || value === undefined || value === '') return '';
  const num = typeof value === 'number' ? value : parseFloat(String(value));
  if (!Number.isFinite(num)) return '';
  const str = (num * 100).toFixed(4);
  return str.replace(/(\.\d*?[1-9])0+$/u, '$1').replace(/\.0+$/u, '');
}
```
