## Technical Notes

### Issues Found During Implementation

1. **Sort indicator in headers:** React Data Table appends `▲`/`▼` to the active sort column header text. Fixed with `cleanHeader()` that strips these characters.

2. **Double space in program name:** The program name in the DB/API contains a double space (`"2018 13 Month Program  (SAC)"`). Fixed with `normalizeSpaces()` that collapses `\s+` to single space for matching.

3. **Column width on parent element:** The `width: '150px'` from the React Data Table config is applied as `min-width`/`max-width` on the parent `rdt_TableCol` element, not on the `[role='columnheader']` element itself. Fixed by traversing to parent with `.locator('..')`.

4. **Pagination (308 programs in STG):** Table shows 10 rows per page by default (max 100). Program edit uses inline form on same `/programs` URL. Fixed `clickProgramLink()` with pagination support + `goToFirstPage()`.

5. **Currency-masked inputs:** `processingFeeOverride` and `amountChargedAtSigning` are text inputs with `$X.XX` mask. Fixed with triple-click select-all before `fill()`.

6. **SAVE button selector:** The composite selector `"button:has-text('SAVE'), button >> span:has-text('SAVE')"` caused Playwright parsing issues. Fixed with `page.getByRole('button', { name: 'SAVE', exact: true })`.
