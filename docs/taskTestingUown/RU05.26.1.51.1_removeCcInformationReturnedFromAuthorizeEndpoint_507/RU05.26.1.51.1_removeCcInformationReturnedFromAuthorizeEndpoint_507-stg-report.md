## Descrição

Hotfix #507 enxuga a resposta do endpoint `submitApplication` (e fluxos correlatos do `/authorizeCreditCard`) para um **slim DTO** com apenas `{status, error, errorCode, preAuthStatus}`. A motivação é remover qualquer vazamento de dados sensíveis do gateway/cartão (`gatewayRequest`, `gatewayResponse`, `token`, `ccNumber`, `creditCardTransactionPk`, etc.) que antes apareciam na resposta. O comportamento funcional (aprovar/recusar pré-autorização, persistir transação) permanece inalterado — só o payload exposto ao cliente foi reduzido.

### Backend ground truth (referência da fix)

`AuthorizeCreditCardResponse` é um Java `record` com **exatamente 4 campos**:

```java
record AuthorizeCreditCardResponse(
    CCTransactionStatus status,
    String error,
    String errorCode,
    PreAuthStatus preAuthStatus
)
```

- `CCTransactionStatus` (enum): `PENDING, FUTURE_PENDING, APPROVED, DENIED, ERROR, REFUNDED, CANCELLED, MANUAL_REVERSE, PICKED_TO_SEND, PARTIALLY_REFUNDED, SKIPPED, REUSED`.
- `PreAuthStatus` (enum): `SESSION_UNAVAILABLE, SUCCESS, DENIED, ERROR, NOT_RUN`.

Em **stg**, `preAuthStatus="NOT_RUN"` porque a feature `is.preauthorization.required.for.origination=false` está desabilitada por configuração de ambiente (Kount preauth desligado). O gateway propriamente dito **está rodando** — `status=APPROVED` no caminho aprovado prova isso.

### Detalhe — bytes do payload

CT-01 e CT-02 confirmam que o slim DTO completo cabe em **77 bytes**, muito abaixo do sanity guard de 500 bytes da SPEC e ordens de grandeza menor que o payload pré-fix (que continha o `gatewayResponse` inteiro).

### Detalhe — Spring error envelope (CT-04 / CT-06)

Quando o backend rejeita um CC malformado **antes** de chegar no controller principal (validação Bean / formato), o handler de exceção do Spring devolve `{message, status}` em vez do slim DTO de 4 campos. **Ambos os formatos passam o helper `assertNoSensitiveLeak`** (deny-list + payload size <300 bytes), então não há vazamento. Documentado como nota técnica nos pitfalls abaixo.
