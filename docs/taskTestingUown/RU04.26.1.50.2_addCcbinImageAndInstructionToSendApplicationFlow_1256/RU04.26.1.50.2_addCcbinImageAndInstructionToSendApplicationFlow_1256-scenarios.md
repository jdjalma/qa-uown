# Test Scenarios — Task #1256 + MR 675

> **Milestone:** RU04.26.1.50.2 (Hotfix)
> **Features:**
> 1. Add CCBIN Image and Instruction to Send Application Flow (#1256 — Origination)
> 2. Edit/Cancel Pending Credit Card Transactions (MR 675 — Servicing)

---

## US-CCBIN-01: Exibir imagem e instrucao do CCBIN no formulario de aplicacao

**Como** um consumidor preenchendo o formulario de aplicacao,
**Quero** ver uma imagem de exemplo e texto explicativo ao lado do campo CCBIN,
**Para** entender onde encontrar os primeiros 6 digitos do meu cartao de credito.

### Criterios de Aceite
- [x] Imagem de exemplo (`ccbin.png`) visivel ao lado do campo CCBIN
- [x] Texto explicativo: "Enter the first 6 digits from the card you used for your payment. This helps us confirm your identity."
- [x] Layout responsivo: 2 colunas desktop, empilhado mobile

---

## US-CCEDIT-01: Editar transacao de cartao de credito pendente

**Como** um operador do Servicing,
**Quero** editar ou cancelar transacoes CC pendentes,
**Para** corrigir erros antes do processamento ou cancelar transacoes indevidas.

### Criterios de Aceite
- [x] Modal "Edit Pending Credit Card Payment" com campos: Posting Date, Amount, Comment
- [x] Botao SAVE atualiza a transacao
- [x] Botao Cancel/Remove altera status para CANCELLED
- [x] Icone de edicao aparece SOMENTE em transacoes PENDING
- [x] Validacoes: campos obrigatorios, amount > 0, posting date >= hoje, comment max 500 chars

**Portal:** Servicing

**Passos:**
1. Login no Servicing portal
2. Navegar para Credit Card History da conta
3. Verificar que icone de edicao (pencil) aparece na linha da transacao PENDING
4. Clicar no icone de edicao
5. Verificar que modal "Edit Pending Credit Card Payment" abre
6. Verificar campos read-only: Reference #, Created Time
7. Alterar Amount para novo valor
8. Alterar Posting Date para data futura valida
9. Preencher Comment: "QA test edit"
10. Clicar SAVE
11. Verificar toast de sucesso
12. Verificar que transacao atualizada na tabela reflete novos valores
13. Validar no DB: `uown_sv_credit_card_transaction` com novo amount, posting_date, comment

**Resultado esperado:**
- Modal abre com dados atuais da transacao
- Apos SAVE: tabela atualiza, DB reflete mudancas
- Status permanece PENDING

**Tags:** @regression @servicing @critical

### CT-05: Edit modal validation — required fields and constraints

**Tipo:** E2E
**Portal:** Servicing
**Pre-condicao:** Conta com transacao CC PENDING existente

**Passos:**
1. Login no Servicing portal
2. Abrir modal de edicao para transacao PENDING
3. Limpar campo Amount → verificar erro "Amount is required."
4. Preencher Amount com 0 → verificar erro "Enter an amount greater than 0."
5. Limpar campo Posting Date → verificar erro "Effective transaction date is required."
6. Inserir data passada → verificar erro "Please enter a valid date."
7. Limpar campo Comment → verificar erro "Comment is required."
8. Preencher Comment com mais de 500 caracteres → verificar erro "Maximum character length is 500."

**Resultado esperado:**
- Todos os erros de validacao exibidos corretamente
- Formulario NAO submete com dados invalidos

**Tags:** @regression @servicing @validation
