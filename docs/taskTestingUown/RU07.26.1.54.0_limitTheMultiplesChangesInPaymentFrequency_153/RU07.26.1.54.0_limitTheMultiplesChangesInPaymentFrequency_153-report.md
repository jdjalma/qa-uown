## O que o qa-debugger corrigiu entre cycle 1 e cycle 2

`src/pages/website/payment-frequency.page.ts` → `pickNextPayday()` passou a injetar `calculateDate()`
(formato `MM/DD/YYYY`, confirmado via DOM real: `<input id="nextPayDate" placeholder="MM/DD/YYYY" type="search">`)
em vez de `calculateDateISO()` (`YYYY-MM-DD`), que corrompia o valor exibido (`"20/26/0706"`) e bloqueava
o SAVE FREQUENCY. `npx tsc --noEmit` limpo.

**Resultado desta correção (confirmado no cycle 2, 2 rodadas independentes — desktop + mobile-android):**
CT-12/CT-13/S2 (Semi-Monthly→Bi-Weekly, campo next-payday) passaram integralmente — o bloqueio
do cycle 1 foi resolvido para o cenário S2 isolado.

## O que o qa-debugger corrigiu entre cycle 2 e cycle 3 (fechamento)

`src/pages/website/payment-frequency.page.ts` → `saveFrequency()` (linha ~276) passou a chamar
`waitForFrequencyRevalidation()` (delega para `BasePage.waitForSpinner()`) logo após
`waitForSuccessToast()`, antes de retornar. Causa raiz confirmada via DOM real (contas sandbox
17358/17359): após o Save, um spinner Bootstrap padrão (`<div class="spinner-border" role="status">`,
exatamente `SELECTORS.spinnerBorder`) renderiza por ~914ms enquanto o card "Current Plan" desmonta e
remonta com o novo valor. `getCurrentFrequency()` chamado logo após o toast podia ler o valor antigo ou
um estado intermediário sem o node — isso quebrava especificamente o 2º save de um ciclo encadeado
(S3). `saveFrequency()` nunca chamava `waitForSpinner()` antes desse fix. `npx tsc --noEmit` limpo
(reverificado por este validator nesta rodada).

**Resultado desta correção — CONFIRMADO de forma independente nesta rodada (cycle 3), não apenas
citado do relato do debugger:** este validator re-executou a suíte completa do zero em `website-ui` e
`website-mobile-android`, e adicionalmente rodou 6 queries `SELECT` read-only diretas nas contas frescas
provisionadas nesta própria execução (17371-17376) para confirmar o resultado no nível de dado, não só
no nível de `reporter=list`. Ver seção "Execução do Teste — cycle 3" e CT-14/S3 abaixo para a evidência
completa coletada por este validator.
