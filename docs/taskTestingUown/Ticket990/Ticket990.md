UOWN | Originação | Recuperação de Log de Atividade Paginada em Ambos os Portais (Originação /Serviço)

Sinopse
Atualmente, o getLogsForLead o endpoint retorna todos os logs de atividade de uma só vez, 
levando a ineficiências de desempenho e longos tempos de carregamento. 
Este bilhete tem como objetivo implementar paginação para recuperação de log em ambos os portais para otimizar 
o carregamento de dados e melhorar a experiência do usuário.

Etapas de Teste:
Verifique se os logs na origem (lead) e na manutenção (conta) estão usando a paginação corretamente e podem selecionar linhas:

Validar se o tempo para os métodos em ambos os portais é reduzido com a paginação

Valide no banco de dados se a resposta de log no frontend corresponder aos resultados nas tabelas: los_activity_log e sv_activity_log


Validar se os métodos funcionam corretamente via api:
- Origination:
curl --location 'https://svc-qa1.uownleasing.com/uown/los/getLogsForLead/574' 
--header 'Content-Type: application/json' 
--header 'Authorization: knQ9MeLBQhxl3gKnOszKq7ZaRxaHY23G2iddtlH2' 
--data '{
"page": 0,
"size": 10
}'

- Servicing:
curl --location 'https://svc-qa1.uownleasing.com/uown/svc/getLogs/60?page=0&size=10' 
--header 'Authorization: knQ9MeLBQhxl3gKnOszKq7ZaRxaHY23G2iddtlH2' 
--data ''
