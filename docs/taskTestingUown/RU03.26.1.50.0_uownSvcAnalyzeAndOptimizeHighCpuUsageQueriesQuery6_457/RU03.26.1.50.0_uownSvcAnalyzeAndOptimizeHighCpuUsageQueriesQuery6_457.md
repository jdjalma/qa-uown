# RU03.26.1.50.0 · Query-6 · uown_los_activity_log (#457)

**GitLab:** https://gitlab.com/uown/backend/svc/-/work_items/457
**Milestone:** Uown | RU03.26.1.50.0

## What was optimized

The query that retrieves the activity history of a lead (`uown_los_activity_log`) was causing high CPU usage due to the lack of an adequate index. The table accumulates all system actions for each lead (status changes, API calls, e-sign events, etc.) and grows continuously.

The optimization added a composite index that allows the system to locate the logs for a specific lead by type and sort by date without scanning the entire table (~576K records in QA).
