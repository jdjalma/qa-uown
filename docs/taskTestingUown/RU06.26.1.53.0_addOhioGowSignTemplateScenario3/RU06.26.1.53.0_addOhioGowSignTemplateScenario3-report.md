## Bugs de Aplicação Encontrados

### BUG-01 — `{{nextPaymentDueAmount}}` em branco no contrato OH GowSign — **FIXED [CONFIRMADO]**

- **Status:** ✅ **CORRIGIDO** e verificado (era CONFIRMADO em 2026-06-18, fix dev antes do release 2026-06-23).
- **Severidade original:** S2 (valor de pagamento ausente no contrato visto pelo cliente) / Prioridade P1.
- **Descrição:** o slot `{{nextPaymentDueAmount}}` renderizava **vazio** em Item 3 e Item 4a do contrato OH GowSign, nas 3 frequências (o `DocumentDispatchService` omitia o token do variables map).
- **Como reproduzir (pré-fix):** aprovar lead OH 16m → dirigir signing → capturar PDF → Item 3/4a em branco.
- **Verificação do fix (2026-06-22), duas fontes primárias independentes:**
  1. **PDF renderizado** — o token agora renderiza o valor concreto: WEEKLY $22.07, BI_WEEKLY $43.50, MONTHLY $95.16 `[render rawText:reports/_rawtext_*.txt]`.
  2. **DB** — a dispatch note "variables map missing [nextPaymentDueAmount]" não aparece mais: `count(*) = 0` `[db-observation:lead_pk IN (16865,16866,16867)]`.
- **Cenário que detectou:** CT-01 (gate de regressão).
- **Causa provável (corrigida):** variables map do dispatch não incluía o token; fix dev passou a popular `nextPaymentDueAmount`.
