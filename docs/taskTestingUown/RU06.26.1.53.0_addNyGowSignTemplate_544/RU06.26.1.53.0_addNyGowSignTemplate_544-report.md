### CT-03 — EPO Promotional-Payoff (epoDays) — Bug Conhecido (AC-3 / AC-4)

**Objetivo:** Validar que a seção de Early Purchase Option renderiza o número de dias (token `epoDays`).

**O que é verificado:** O contrato deve exibir "Purchase the Property within N days" ou "During the first N days from the date of your Initial Payment" (onde N é o valor de `epoDays`). O Signwell renderiza "within 90 days". O GowSign renderiza em branco ("within  days") porque o backend (`DocumentDispatchService`) não envia o token `epoDays` para o mapa de variáveis GowSign — confirmado em log de backend e cross-verificado com svc#555 (Alabama, mesma regressão). `[db-observation:uown_los_lead_notes lead_pk=16684]` `[skill:gowsign-knowledge BR-06]`

#### Como verificar manualmente
1. Abrir contrato GowSign NY e localizar a seção "Promotional Payoff Option" / "EARLY PURCHASE OPTIONS".
2. Ler a frase "Purchase the Property within ___ days" — o campo deve estar em branco.
3. Comparar com qa1 (Signwell): "within 90 days" aparece preenchido.
4. SQL: `SELECT notes FROM uown_los_lead_notes WHERE lead_pk=<pk> AND notes ILIKE '%missing%token%epoDays%'` — log de backend confirma token ausente.

**PASSOU (expected-failure)** — Teste marcado `test.fail(true, 'svc#544 BR-06: epoDays token not populated by backend')`. Comportamento confirmado: EPO renderiza em branco. Quando o dev corrigir o backend, o teste "passará inesperadamente" (`unexpected pass`) e o `test.fail` deve ser removido. `[test-execution:btnqqtf2z Test 3,2026-06-18]`

## Bugs de Aplicação Encontrados

### BUG-01 — epoDays não enviado pelo backend para o template GowSign NY

| Campo | Valor |
|---|---|
| Status | **ABERTO** |
| Severidade | Medium (conteúdo contratual incompleto — EPO em branco para o cliente) |
| Afeta | GowSign NY (`NY_2025_SAC`) — mesma regressão confirmada em AL (`AL_2025_SAC`, svc#555) |
| Descoberto | 2026-06-17, svc#544 discovery |
| Cenário que detectou | CT-03 |

**Descrição:** O `DocumentDispatchService` do backend não inclui o token `epoDays` no mapa de variáveis enviado ao GowSign ao criar o documento. O template NY tem o placeholder `{{{epoDays}}}` na seção "Promotional Payoff Option" que renderiza em branco, exibindo "Purchase the Property within  days" (sem número). O Signwell renderizava "within 90 days" com o valor correto.

**Como Reproduzir:**
1. Criar lead NY TireAgent em qa2 → contrato GowSign gerado.
2. Abrir o contrato e localizar a seção "EARLY PURCHASE OPTIONS" → "Promotional Payoff Option".
3. Verificar que o número de dias está em branco.
4. SQL: `SELECT notes FROM uown_los_lead_notes WHERE lead_pk=<pk> ORDER BY pk` — buscar log `[DocumentDispatchService][GowSign] missing ... [epoDays]`.

**Evidência:** `[test-execution:btnqqtf2z Test 3,2026-06-18]` `[skill:gowsign-knowledge BR-06]`

**Causa Provável:** `DocumentDispatchService.buildGowSignVariables()` (ou equivalente) não mapeia `epoDays` para o payload de criação do documento GowSign, ao contrário do Signwell que recebia esse valor. Mesma causa raiz da regressão em AL (svc#555).
