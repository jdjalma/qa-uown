## Descrição

Migração da assinatura de contrato de **Signwell** (legado) para **GowSign** (novo provider) para o estado de **New York**. QA deve: (1) validar comportamento do template assinando pelo fluxo de estado correto; (2) comparar conteúdo do contrato contra o Signwell existente; (3) garantir valores calculados, variáveis e lógica de template corretos; (4) validar que dados dinâmicos populam corretamente no GowSign. Cenários do ticket: (1) Origination portal, (2) Partner portals PayTomorrow/TireAgent, (3) Contract validation vs Signwell, (4) State validation. Template criado: `NY_2025_SAC` (pk=16, qa2, 13m; sem template 16m para NY).

### CT-02 — Content Parity vs Signwell + Value-for-Value (AC-2 / AC-3 / Scenario 3)

**Objetivo:** Confirmar que o contrato GowSign NY mantém todas as cláusulas do contrato Signwell NY legado e que cada valor calculado pelo backend é renderizado no corpo do documento.

**O que é verificado:**
- Paridade de conteúdo: 15 cláusulas do baseline Signwell (`PROPERTY PRICE TAG`, `TOTAL OF PAYMENTS`, `COST OF LEASE`, `CASH PRICE`, `Description of the Property`, `Promotional Payoff Option`, `EARLY PURCHASE OPTIONS`, `Early Purchase Option`, `Returned Payment`, `Late Fee`, `Reinstatement`, `ACH Payment Authorization`, `Income Interruption Rights`, `Electronic Signatures`, `Mollie, LLC`) presentes no GowSign.
- Título "RENTAL-PURCHASE AGREEMENT-NY" (paridade com Signwell — NY usa "Rental-Purchase", não "Lease-Purchase").

`[test-execution:btnqqtf2z Test 2,2026-06-18]` `[db-observation:uown_esign_document.request lead_pk=16683]`

### CT-03 — EPO Promotional-Payoff (epoDays) — Regressão CORRIGIDA (AC-3 / AC-4)

**Objetivo:** Validar que a seção de Early Purchase Option renderiza o número de dias (token `epoDays`).

**O que é verificado:** O contrato deve exibir "Purchase the Property within N days" ou "During the first N days from the date of your Initial Payment" (onde N = `epoDays`). O Signwell renderiza "within 90 days". **Em 2026-06-18 o GowSign renderizava em branco** ("within  days") porque o backend (`DocumentDispatchService`) não enviava o token. **Corrigido para o R1.53.0:** em 2026-06-22 o contrato GowSign NY renderiza "within **90** days" — paridade com o Signwell. `[test-execution:2026-06-22 Test 3]` `[db-observation:uown_esign_document.request lead_pk=16812 epoDays=90]` `[skill:gowsign-knowledge BR-06]`

#### Como verificar manualmente
1. Abrir contrato GowSign NY e localizar "Promotional Payoff Option" / "EARLY PURCHASE OPTIONS".
2. Ler "Purchase the Property within ___ days" / "During the first ___ days" — deve mostrar **90**.
3. Comparar com Signwell ("within 90 days") — agora idênticos.
4. SQL: `SELECT request::text FROM uown_esign_document WHERE lead_pk=<pk> AND document_name='NY_2025_SAC'` → variables map contém `epoDays="90"`; NÃO há mais log `[DocumentDispatchService][GowSign] … missing … [epoDays]`.

**PASSOU (pass real)** — O `test.fail` foi removido; o teste agora é uma asserção positiva (`Purchase the Property within \d+ days` / `During the first \d+ days…`). Lead 16884, 28.4s. `[test-execution:2026-06-22 Test 3]`

## Bugs de Aplicação Encontrados

### BUG-01 — epoDays não enviado pelo backend para o template GowSign NY — ✅ RESOLVIDO

| Campo | Valor |
|---|---|
| Status | **RESOLVIDO** (corrigido no R1.53.0, verificado em qa2 2026-06-22) |
| Severidade | Medium (era: conteúdo contratual incompleto — EPO em branco para o cliente) |
| Afeta | GowSign NY (`NY_2025_SAC`) — mesma família de regressão de AL (svc#555) |
| Descoberto | 2026-06-17 (discovery) · Corrigido | 2026-06-22 |
| Cenário que detectou / guarda | CT-03 |

**Descrição (histórica):** O `DocumentDispatchService` não incluía o token `epoDays` no mapa de variáveis enviado ao GowSign. O placeholder `{{{epoDays}}}` na seção "Promotional Payoff Option" renderizava em branco ("Purchase the Property within  days"), enquanto o Signwell renderizava "within 90 days".

**Resolução (2026-06-22):** o backend agora popula `epoDays`. Evidência primária: lead fresh **16812** (`uown_esign_document.request` variables map) contém **`epoDays="90"`**, e **nenhum** lead NY de 2026-06-21/22 emite o log `[DocumentDispatchService][GowSign] … missing … [epoDays]` (contraste: lead 16684 de 2026-06-18 tinha o token ausente + o log). CT-03 confirma o render: o contrato exibe "within 90 days". O `test.fail` foi removido — CT-03 é um guarda de regressão positivo permanente.

**Evidência:** `[test-execution:2026-06-22 Test 3]` `[db-observation:uown_esign_document.request lead_pk=16812 epoDays=90]` `[skill:gowsign-knowledge BR-06]`
