## Descrição

Antes deste hotfix, uma linha de pagamento `STICKY`/`PAID` no Payment History do Servicing oferecia apenas a opção **Fully Refund** no modal de Reverse Payment — a opção **Reverse** (ledger-only, sem devolução via Sticky) não estava disponível para pagamentos recuperados via Sticky, mesmo já existindo como ação genérica para pagamentos CC/ACH não-Sticky.

A ticket solicita adicionar a opção **Reverse** ao conjunto de Reverse Reason de uma linha Sticky/PAID, para que o agente de Servicing possa corrigir um registro de pagamento recuperado via Sticky sem precisar sempre devolver o dinheiro pelo Sticky. O comportamento existente de **Fully Refund** em Sticky deve permanecer inalterado, e o comportamento de reverse/refund para pagamentos não-Sticky (CC/ACH) não pode regredir.
