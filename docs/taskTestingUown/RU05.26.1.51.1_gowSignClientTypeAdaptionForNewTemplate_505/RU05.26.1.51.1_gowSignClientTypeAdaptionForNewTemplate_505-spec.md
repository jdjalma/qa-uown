## Edge Cases

- **Daniel's `integration_type=HYBRID`:** submit/dispatch path may differ from non-hybrid merchants. - **CT-11 brand mismatch:** Kornerstone account creation may not auto-set `company='KORNERSTONE'` if merchant config is incomplete. STOP and request user UPDATE auth — do NOT silently UPDATE (Exception 3).

- **`pdf-parse` install:** new dep — require explicit user authorization in Phase 5 (do not auto-install).
- **CT-05 merchant TBD:** if zero CA UOWN non-jewelry merchants returned by query, escalate to user before fabricating data. Do NOT pick Kornerstone substitute (different brand).
- **Activity log absence:** rule #14 makes absence a `[CONFIRMADO]` violation; but only if log is reasonably expected for that action type. PATCH template (CT-08) audit log may not be required by product if there's no audit table — document explicitly in report.
