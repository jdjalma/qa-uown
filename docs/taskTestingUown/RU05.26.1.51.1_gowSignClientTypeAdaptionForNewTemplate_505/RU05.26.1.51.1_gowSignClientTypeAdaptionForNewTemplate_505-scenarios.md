## Resumo da feature

GowSign template selection passou a depender do **`clientType`** do merchant além do `state`. Antes: `state → templateId`. Agora: `(state, clientType) → templateId`, com fallback para templates onde `client_type IS NULL/blank`.

**Mudanças backend (R1.51.1):**
- `uown_gow_sign_template.client_type` (TEXT, comma-separated) — adicionado V20260430055536, ampliado p/ TEXT em V20260505041814
- `SqlTemplateSelectionService.findForCustomerStateAndClientType(state, clientType)`
- `GowSignTemplateRepo.findEligibleByStateAndClientOrdered(state, clientType)` — match por state + parsing CSV de clientType com fallback a rows blank; ordering: explicit-match > rowUpdatedTimestamp DESC > pk DESC
- `DocumentOrchestrator.createDocument(...)` agora lê `merchant.getMerchantInfo().getClientType()` (enum) → `Enum::name()` → passa string ao selector
- Novo `PATCH /uown/svc/gowsign-templates/{templateId}` (`GowSignTemplatePatchInfo` — campos opcionais)

**ClientType enum (relevante p/ esta task):**
- `JEWELRY` (user `uownJewellers`, merchant ref code `uownJewellers`)
- `DANIELS_JEWELERS` (Daniel's Jewelers, `integration_type=HYBRID`)

**Templates novos (CA):**
- Template CA + clientType=JEWELRY ou DANIELS_JEWELERS → conteúdo "Rental" (não "Lease"), security deposit, jurisdiction/limitation of liability sections, delivery fee text alternativo
- Template CA padrão (clientType vazio) → conteúdo "Lease", sem security deposit

## User Stories

### US-GS505-01: Como Compliance Officer

**Como** Compliance Officer da UOWN,
**Quero** que clientes de merchants do segmento "jewelry" (DANIELS_JEWELERS, JEWELRY) recebam um contrato com terminologia *Rental* + cláusulas de jurisdição/security deposit específicas para a CA,
**Para** atender ao requisito legal de jewelry leases na Califórnia.

**Critérios de Aceite**
- [ ] Contratos gerados para merchant `clientType=DANIELS_JEWELERS` em CA usam o template específico (não o template CA padrão)
- [ ] Contratos gerados para merchant `clientType=JEWELRY` em CA usam o template específico
- [ ] Contratos para merchant `clientType` qualquer outro (ou vazio) em CA continuam usando o template CA padrão (regressão)
- [ ] Templates novos contêm: Security Deposit, terminologia "Rental", delivery fee alternativo, Jurisdiction, Limitation of Liability

### US-GS505-02: Como Customer Support

**Como** agente de Customer Support no portal Servicing,
**Quero** que o lease document do cliente apareça na seção Documents do customer details e seja baixável,
**Para** poder reenviar o contrato sob demanda.

**Critérios de Aceite**
- [ ] Após sweep `getCompletedESignDocumentStatusSweep`, o documento aparece na Documents section
- [ ] Click em download retorna o PDF assinado
- [ ] PDF contém os mesmos placeholders renderizados (nome, valores, datas)

### US-GS505-03: Como Backend Engineer

**Como** Backend Engineer,
**Quero** atualizar template metadata via `PATCH /uown/svc/gowsign-templates/{templateId}` sem reescrever o registro inteiro,
**Para** não impactar outros campos durante hotfix.

**Critérios de Aceite**
- [ ] PATCH com body parcial (ex: só `clientType`) atualiza apenas o campo enviado, mantendo os demais
- [ ] PATCH valida que `templateId` existe — 404 se não
- [ ] PATCH normaliza `clientType` (uppercase + remove whitespace) — confere com `GowSignTemplateService`
- [ ] Após PATCH, lookup `findEligibleByStateAndClientOrdered` reflete a mudança

### CT-03_DocumentContentDifferentialJewelryVsStandard_CA

**Tipo:** Hybrid (API gera + UI/PDF inspeciona)
**Portal:** Origination embedded modal + GowSign document viewer
**Pré-condições:** dois leads em paralelo:
- Lead A: `DanielsJewelers` (clientType=DANIELS_JEWELERS) em CA
- Lead B: merchant CA UOWN não-jewelry (ex: `TerraceFinance`) em CA

**Passos:**
1. Gerar contrato GowSign para Lead A e Lead B (em paralelo via `Promise.all`)
2. Abrir `GowSignDocumentViewerPage` (ou capturar HTML do iframe / extrair PDF) para cada lead
3. Para o **Lead A (jewelry template)** validar presença de:
   - Texto contendo `Security Deposit` + valor `{{securityDeposit}}` renderizado (não literal)
   - Terminologia `Rental` em headings (`Rental Period`, `Cost of Rental`)
   - Texto `*Includes a Delivery Fee of` (não `*Total Delivery Fee`)
   - Section `Jurisdiction` presente
   - Section `Limitation of Liability` presente
   - Texto `Cash Price of the item(s)` na cláusula de Early Purchase
4. Para o **Lead B (standard CA template)** validar:
   - **Ausência** de section `Security Deposit`
   - Terminologia `Lease` (não `Rental`) em headings
   - Texto `*Total Delivery Fee`
   - Texto `Cash Price` (sem `of the item(s)...` extra) na Early Purchase
5. Cross-check de não-contaminação: jewelry template NÃO contém marcadores Kornerstone; standard CA template NÃO contém `Rental`

**Resultado esperado:** ambos templates renderizam variáveis (sem `{{...}}` literais) e os 5 marcadores diferenciam corretamente.

> **Nota de implementação:** usar `gowsign-signing.helper.ts` p/ acesso ao iframe; ou baixar o PDF e inspecionar texto via `pdf-parse`. Fallback: comparar contra o HTML body do iframe.
