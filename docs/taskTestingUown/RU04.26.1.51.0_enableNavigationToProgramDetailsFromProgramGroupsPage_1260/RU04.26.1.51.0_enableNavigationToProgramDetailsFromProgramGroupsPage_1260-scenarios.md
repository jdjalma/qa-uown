# Test Scenarios — Enable Navigation to Program Details from Program Groups Page (#1260)

**Task:** https://gitlab.com/uown/frontend/origination/-/work_items/1260
**Portal:** Origination
**Environment:** qa2
**URL:** https://origination-qa2.uownleasing.com/programGroups
**MRs:** Frontend !1429, Backend svc!1369, Backend svc!1368 (ACH fix)

---

## US-PG-01: Navigate to Program Details from Program Groups Modal

**Como** usuário do Origination portal,
**Quero** clicar no nome de um programa dentro do modal de Program Groups,
**Para** ser redirecionado para a página de detalhes do programa correspondente.

### Critérios de Aceite
- [ ] No modal de Program Groups, cada programa exibe nome como link clicável
- [ ] Clicar no nome navega para `/programs/{pk}?from=programGroups`
- [ ] A página de Program Details carrega com os dados corretos do programa
- [ ] O botão Cancel na página de detalhes volta para `/programGroups`

---

## US-PG-02: Display Program Details Table in Groups Modal

**Como** usuário do Origination portal,
**Quero** ver uma tabela detalhada com informações dos programas ao abrir um grupo,
**Para** ter visibilidade dos dados financeiros antes de navegar.

### Critérios de Aceite
- [ ] Modal exibe tabela com colunas: Program Name, Term Months, Money Factor, EPO Days, Processing Fee, Pay Off Discount
- [ ] Dados são carregados da API `GET /uown/programs/groups` (novo formato JSON)
- [ ] Tabela mantém funcionalidade existente de exibir programas do grupo

---

## US-PG-03: Preserve Existing Group Behavior

**Como** usuário do Origination portal,
**Quero** que o comportamento existente da página de Program Groups continue funcionando,
**Para** não ter regressão nas funcionalidades já existentes.

### Critérios de Aceite
- [ ] A página carrega a lista de grupos
- [ ] A busca/filtro de grupos funciona
- [ ] A edição de nome do grupo funciona
- [ ] O contador de programas por grupo está correto

### CT-06: API GET /uown/programs/groups returns enriched program data
**Tipo:** API
**Portal:** Origination (backend svc)
**Pré-condição:** Auth token válido

**Passos:**
1. Chamar `GET /uown/programs/groups` via API
2. Validar estrutura da resposta

**Resultado esperado:**
- Response 200
- Cada grupo contém: `groupName` (string), `count` (number), `programs` (array)
- Cada programa em `programs` contém: `pk` (number), `programName` (string), `termMonths` (number), `moneyFactor` (number), `epoDays` (number), `processingFee` (number), `payOffDiscount` (number)
- `count` == `programs.length` para cada grupo

**Tags:** @regression @api

## Mapeamento QA Domain Reflexes

| Ação | Reflexo Aplicável | Validações |
|------|-------------------|------------|
| Navegação entre páginas | Nenhum bloco específico → **11. Qualquer Mutation (fallback)** | URL correta, dados carregados, sem erros console |
| API retorna dados | **11. Qualquer Mutation** | Estrutura de resposta, campos obrigatórios presentes |
| Cancel/back navigation | Nenhum bloco específico | Retorno à página correta |

**Nota:** Esta tarefa é predominantemente de **navegação e apresentação de dados** (read-only). Não há mutations de estado, pagamentos, ou transições de status. Os reflexos de domínio aplicáveis são mínimos (bloco 11 como fallback).
