Implementar navegação do modal de Program Groups para a página de Program Details. ### CT-09 — Program details data matches modal data

**Objetivo:** Verificar que os dados do programa exibidos no modal são consistentes com os dados na página de Program Details.

**Resultado em stg:** **FALHA DO TESTE (não é bug de aplicação)**

**Detalhe do assert que falhou:**

```
Step: "Verify Money Factor matches"
expect(formMoneyFactor).toBe(modalMoneyFactor)

Expected (modal): "17.306"
Received (form):  "17.305999999999997"

  at spec.ts:494
```

**Conclusão: NÃO é bug.** Os dois valores representam o **mesmo número** matemático — `17.305999999999997` é apenas a representação exata em double IEEE-754 que, ao ser arredondada para 3 casas decimais, vira `17.306`. Comportamento esperado de ponto flutuante.

A causa da falha é o **assert do próprio teste**: o spec usa igualdade exata de string (`toBe`) para comparar dois valores que são o mesmo número renderizado de formas diferentes pela aplicação (modal aplica formatação `.toFixed(3)` na exibição; o input do form expõe o valor raw do estado). Em qa2 esse cenário não disparou apenas por sorte: o programa amostrado lá (no grupo `CloneAllGroup`) tinha um `moneyFactor` representável exatamente em float, sem cauda binária. Em stg, o grupo amostrado (`"2025 10% EPO"`) tem `moneyFactor` que expõe a representação binária, fazendo o `toBe` falhar — comportamento idêntico ao observado na execução anterior em 2026-04-27.

**Classificação:** falha de teste por assert frágil, não defeito de aplicação. A funcionalidade está OK em stg — o programa exibido no Details corresponde ao do modal; a única diferença é o nível de formatação visual entre as duas telas, e isso é decisão de UX, não bug.

**Ação recomendada (no spec, não no app):** trocar `toBe` por comparação tolerante a precisão de float — por exemplo `expect(parseFloat(formMoneyFactor)).toBeCloseTo(parseFloat(modalMoneyFactor), 3)` ou normalizar via `parseFloat(x).toFixed(3)` antes do `toBe`. Fora do escopo desta execução.

#### Como verificar manualmente

1. Acessar `https://origination-stg.uownleasing.com/programGroups`.
2. Abrir modal do grupo "2025 10% EPO".
3. Anotar Program Name e Money Factor do primeiro programa (Money Factor exibido como `17.306`).
4. Clicar no nome do programa para navegar ao Details.
5. Comparar Program Name (igual) e Money Factor — no input aparece `17.305999999999997`, que é o mesmo `17.306` em representação float não formatada. Funcionalmente equivalente.

**FALHOU (assert frágil — não é bug)**
