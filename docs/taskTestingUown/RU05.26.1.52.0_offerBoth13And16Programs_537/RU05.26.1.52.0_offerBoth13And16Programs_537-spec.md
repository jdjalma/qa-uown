## 2. Problem Statement

Quando o BlackBox retorna `EligibleTerms="16"` (cliente elegível para 16m, sem lookahead), o produto deve oferecer AMBOS os programas 13m e 16m no fluxo do cliente, em vez de apenas 16m. O fix em `LeadProgramService.getLTOProgramsForLead` faz lookup do novo config `defaultMonthsWhen16Month` (default `"13,16"`) e expande os terms quando `EligibleTerms` contém "16" E `IsEligibleForExtraInfo != true` (guard impede expand no caso "lookahead").

**Matriz pós-fix (referência):**

| EligibleTerms (BlackBox) | IsEligibleForExtraInfo | terms após guard | Result |
|---|---|---|---|
| `"13"` | false | `"13"` | só 13m |
| `"16"` | false | **`"13,16"`** (expand) | **13m + 16m** mudança |
| `"13,16"` | false | `"13,16"` (no-op) | 13m + 16m |
| `""` | true | `"16"` (lookahead) | só 16m (guard impede expand) |
| `""` | false | `""` | fallback `defaultTo13MonthProgram` 13m |
