## Resumo

O fix em `LeadProgramService.getLTOProgramsForLead` expande `EligibleTerms="16"` para `"13,16"` via config `defaultMonthsWhen16Month`, garantindo que clientes elegiveis para 16m visualizem ambos os programas (13m e 16m) na tela de selecao de pagamento. A validacao confirmou tres comportamentos criticos: (1) o componente MissingPaymentProgram renderiza duas tabs com as frequencias corretas por programa, (2) a regressao para clientes 13m-only permanece intacta, e (3) a selecao de planId 13m em lead 16m-elegivel resolve corretamente sem erro 500.

Todos os cenarios executados passaram na validacao obrigatoria de activity log (regra #13), confirmando que o backend persiste o log da decisao de programa em `uown_los_lead_notes`.

## Cenarios validados

### Requisitos da tarefa

#### 1. Cliente 16m-elegivel visualiza tabs 13m e 16m com frequencias corretas

> ✅ **PASSOU** - KS3015 (FifthAveFurnitureNY, Kornerstone) / CA / income 70000

Aplicacao criada via API com income alto e bankData (routing Kornerstone). Ao navegar para a URL do componente MissingPaymentProgram (sem planId), duas tabs sao renderizadas: "13 Months" com 2 cards de frequencia (Weekly, Bi-Weekly) e "16 Months" com 3 cards (Weekly, Bi-Weekly, Monthly). O DB confirma `approved_terms="13,16"` e `uw_terms="16"` em `uown_lead_approval_terms`. O log `After defaulting to 13,16 terms are : 13,16` foi encontrado em `uown_los_lead_notes`.

**Evidencia:** leadPk=12028, approved_terms="13,16", uw_terms="16", log source=lead_notes

#### 2. Regressao EligibleTerms="13" mostra apenas 13m

> ✅ **PASSOU** - KS3015 / CA / income 25000 / sem bankData (routing UOWN)

Aplicacao criada sem bankData para forcar routing UOWN, resultando em EligibleTerms="13". O componente renderiza apenas opcoes de 13m (sem tab 16m no DOM). O DB confirma `approved_terms="13"` e `uw_terms="13"`. O log do `[LeadProgramService][getLTOProgramsForLead]` com `Found program(s) eligible for term 13` foi encontrado em `uown_los_lead_notes`.

**Evidencia:** leadPk=12029, approved_terms="13", uw_terms="13", log source=lead_notes

#### 3. planId 13m em lead 16m-elegivel resolve sem erro 500 (AC5 - ancora)

> ✅ **PASSOU** - KS3015 / CA / income 70000 / planId=WK13

Lead 16m-elegivel (approved_terms="13,16") seleciona o programa 13m via planId WK13. A chamada `getMissingFields(shortCode, { planId: WK13 })` retorna HTTP 200, confirmando que `resolveAndSetMerchantProgramFromPlanId` resolve corretamente o `merchant_program_pk` para o programa 13m. O DB mostra `merchant_program_pk` associado a programa com `term_months=13`. O log de expansao de termos esta presente em `uown_los_lead_notes`.

**Evidencia:** leadPk=12030, getMissingFields=200, merchant_program_pk resolvido para 13m, log source=lead_notes
