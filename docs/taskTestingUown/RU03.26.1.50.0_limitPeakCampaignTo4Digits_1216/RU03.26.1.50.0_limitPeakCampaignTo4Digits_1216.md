**CT-01: Fields reject non-numeric characters on Merchant Settings**

Acessa a pagina Merchant Settings e tenta digitar valores nao numericos nos campos `peakCampaignId` e `offPeakCampaignId`. Para cada campo, digita caractere por caractere os 4 payloads de teste e verifica que somente a parte numerica e aceita:

| Input | Valor esperado no campo |
|-------|------------------------|
| `abcd` | `""` (vazio) |
| `12ab` | `"12"` |
| `12@3` | `"123"` |
| `!@#$` | `""` (vazio) |

O comportamento e controlado pela prop `isNumbersOnly` do componente React, que bloqueia caracteres nao numericos no nivel do keypress.

**PASSED**

**CT-02: Fields enforce a maximum of 4 digits on Merchant Settings**

Acessa a pagina Merchant Settings e testa o limite maximo de 4 digitos em ambos os campos. Primeiro confirma que valores de 1 a 4 digitos sao aceitos integralmente, depois verifica que valores maiores sao truncados:

| Campo | Input | Valor esperado |
|-------|-------|----------------|
| `peakCampaignId` | `1`, `12`, `123`, `1234` | Aceitos integralmente |
| `peakCampaignId` | `12345` (5 digitos) | Truncado para `1234` |
| `offPeakCampaignId` | `99999` (5 digitos) | Truncado para `9999` |
| `offPeakCampaignId` | `1234567890` (10 digitos) | Truncado para `1234` |

A truncagem e feita pelo atributo HTML `maxLength={4}` no DOM.

**PASSED**
