            UOWN | Origination | Add “New Merchant” Log Entry and Log Type Filter Enhancement in Merchant Modification History


            Synopsis
            Currently, in the MERCHANT MODIFICATION HISTORY page of the Origination Portal, users can view modifications made to Merchants in log format. However, there is no specific log entry when a new Merchant is created, making it harder to track these additions.

            Business Objective
            The objective is to improve visibility and control over the Merchant lifecycle. Adding a log whenever a new Merchant is created will provide immediate awareness to users and business teams, ensuring greater transparency and facilitating historical analysis.

            Feature Request | Business Requirements:

            Implement an automatic log entry whenever a new Merchant is added to the system.


            Add a Log Type field to the existing filter panel on the Merchant Modification History page.


            Display the value “New Merchant” in the Log Type column to identify this type of log.


            Enable the Log Type filter to search specifically for “New Merchant” entries.


            Ensure that all existing Log Types can also be searched and displayed through this new filter field.
