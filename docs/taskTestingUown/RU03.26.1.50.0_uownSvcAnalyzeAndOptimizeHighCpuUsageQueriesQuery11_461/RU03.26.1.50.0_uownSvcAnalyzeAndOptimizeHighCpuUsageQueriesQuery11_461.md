## What was optimized

The `getLeadSummaryResults` query assembles the lead summary for display in the Origination portal's Leads table. It joins phone, approval status, address, and lead history data to produce the list shown when agents search for leads. Without adequate indexes, every lookup triggered full table scans on `uown_los_phone`, `uown_los_uwdata`, `uown_los_lead`, and `uown_los_address`, generating CPU spikes.

The optimization added 5 indexes:
- **`idx_phone_full_number_active`** ŌĆö expression index on `(area_code || phone_number)` for phone number concatenation lookups
- **`idx_uwdata_status_expiration`** ŌĆö composite on `(uw_status, approval_expiration_date)` for underwriting status filtering
- **`idx_lead_timestamp_status`** ŌĆö composite on `(row_created_timestamp, lead_status)` for date+status range scans
- **`idx_address_zip_code`** / **`idx_address_state`** ŌĆö single-column indexes for address filtering

Four of these indexes are shared with Query-1 (`getDataMismatchForLead`, Task #449). The expression index `idx_phone_full_number_active` is specific to this query.
