Uown | SVC | Integrate TaxCloud Address Verification Feature

Synopsis
We are seeing some new fraud activity from merchants that we need to address in the UW flow.  What is happening is the customer is loading whole address into the address 1 field in the hopes that with the wrong information on the  account they will get a call from FedEx and can get the delivery.
This is because FedEx does the initial route off of the Zip. 

Business Objective
Business wants to avoid incurring losses because of fraudulent activities with deliveries

Feature Request | Business Requirements

Validate the customer address fields for regular expressions


Attachment(s)
Email communication: Address_validation.pdf

Testing Steps
These changes introduce address verification in the lease creation flow.
1. Test with a fake (invalid) address
Run sendApplication using a sample fake address.
Expected result:
The application is declined.
Message displayed: "No matches were found for address."

2. Test with a valid (production) address
Run sendApplication using a valid production address.
Expected result:
The application passes successfully.
In the los_address table, a new column zipcode9 is persisted.

3. Test ZipCode9 in Carts and Orders Requests
These tests confirm that zipcode9 is now used instead of the standard zip code in Carts (tax evaluation) and Orders (payment sync with TaxCloud).
3.1 Test in Carts
Perform actions that trigger tax evaluation (creating a new application or adding a credit card, etc).
Expected result:
The request to TaxCloud uses the zipcode9 value instead of the standard zip.

3.2 Test in Orders
Perform actions that trigger payment sync with TaxCloud (making a payment in servicing and running /triggerScheduledTask/dailyTaxCloudPaymentsSync).
Expected result:
The request sent to TaxCloud includes zipcode9.
Confirm this in the taxcloud_outbound table by reviewing the request column.

Logs in grafana to check the workflow
[TaxCloudClient][postVerifyAddressRequest] — confirms the request was sent to the external API.
[VerifyAddressService][verifyAddress] — confirms the response was received and zipcode9 was populated.
[TaxCloudService][verifyAddress] — confirms address verification was completed.
[LosRequestMessageConstraintValidator] — validates that constraint checks were applied.
