## Descricao

O Money Factor e a taxa de lease do programa, armazenada no banco como decimal (ex: `0.1846`). Para o usuario, esse valor e exibido multiplicado por 100 com o icone `%` — ou seja, `18.46%`. Antes do fix da #1251, a coluna "Money Factor" na tabela de Programs exibia o valor bruto do banco (`0.1846`) enquanto o formulario de edicao ja exibia corretamente (`18.46`). O fix padronizou a tabela para usar a mesma funcao de conversao `toMoneyFactorDisplayValue()` ja utilizada no formulario:

```
toMoneyFactorDisplayValue(0.1846)
= trimTrailingZeros((0.1846 * 100).toFixed(4))
= trimTrailingZeros("18.4600")
= "18.46"
```

Exemplos de conversao:

| DB `money_factor` | O usuario ve/digita |
|-------------------|---------------------|
| `0.003` | `0.3` |
| `0.169999` | `16.9999` |
| `0.15` | `15` |
| `0.1846` | `18.46` |

## Bugs de Aplicacao Encontrados

### BUG-01 — Campo Money Factor no formulario exibe `18.459999999999997` em vez de `18.46`

| Campo | Valor |
|-------|-------|
| **Ambiente** | qa2 |
| **Descoberto em** | 2026-04-02 — CT-02 |
| **Severidade** | Media — usuario ve valor com ruido de ponto flutuante ao abrir programa existente |
| **Escopo** | Em escopo — task #1251 exige padronizacao do display em todos os locais |

**Descricao:** Ao abrir a pagina de detalhes de um programa, o campo "Money Factor" exibe `18.459999999999997` em vez de `18.46`. A tabela exibe corretamente `18.46`. O usuario que alterna entre a listagem e o formulario ve valores diferentes para o mesmo programa.

**Causa raiz:** Duas funcoes em `utils/money-factor.ts` com comportamentos diferentes ao multiplicar por 100:

```typescript
// Tabela — usa toMoneyFactorDisplayValue() ✅
trimTrailingZeros((0.1846 * 100).toFixed(4))  →  "18.46"
//                              ^^^^^^^^^^
//                              .toFixed(4) controla a precisao antes do trim

// Formulario — usa toMoneyFactorFormValue() ❌
0.1846 * 100  →  18.459999999999997
// multiplicacao direta em JavaScript expoe imprecisao IEEE 754
```

**Fix sugerido em `origination/utils/money-factor.ts`:**
```typescript
// antes
export const toMoneyFactorFormValue = (value: number) => value * 100;

// depois
export const toMoneyFactorFormValue = (value: number) =>
  parseFloat((value * 100).toFixed(4));
```

**Impacto:** Campo `input[name='moneyFactor']` em `/programs/:pk` e `/programs` (lista) exibe valor com ruido. Se o usuario salvar sem alterar o campo, o backend recebe `18.459999999999997 / 100 = 0.18459999999999997` em vez de `0.1846`, podendo causar drift de precisao acumulado em saves sucessivos.
