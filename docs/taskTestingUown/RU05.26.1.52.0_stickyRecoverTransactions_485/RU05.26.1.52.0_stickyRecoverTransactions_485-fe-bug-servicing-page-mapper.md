## Causa raiz real

**Race condition no teste**, não no produto:

1. `loadCreditCardHistory(accountPk)` chama `getCCTransactions` + `getStickyRecoveries` em `Promise.all`
2. `getCCTransactions` resolve rápido → MobX popula `creditCardTransactions` → grid renderiza rows com cells `—` (fallback)
3. `getStickyRecoveries` resolve depois → MobX popula `stickyRecoveriesByCcPk` → re-render preenche cells
4. Teste anterior assertava cells **entre os passos 2 e 3** → captura `—`

## Fix aplicado

- `src/pages/servicing/credit-card-history.page.ts` — novos helpers `waitForStickyRecoveriesResponse()` + `waitForStickyCellPopulated()`
- `tests/e2e/RU05.26.1.52.0_stickyRecoverTransactions_485-ui.spec.ts` — arma `waitForResponse` ANTES da navegação, e depois polla até cell deixar de ser `—`

## Lição

- **Sempre validar a hipótese de bug de produto via endpoint direto** antes de redigir doc de bug externo. (Em 2026-05-20 escrevi um doc de bug FE pra time externo sem ter validado o payload do endpoint primeiro.)
- **`getCCTransactions` vs `getStickyRecoveries` é assíncrono paralelo** — qualquer teste que dependa do enriquecimento Sticky precisa aguardar explicitamente.
