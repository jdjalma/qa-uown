## 1. Synopsis

A task altera o valor padrao da flag "Funding On Hold" no fluxo de criacao de novos Merchants em
Origination (ADD NEW COMPANY). Antes da fix, a flag era criada com `true` por default, colocando
novos merchants em hold de funding desnecessariamente. Apos a fix, o valor padrao deve ser `false`.

Merchants existentes nao devem ser afetados. O usuario ainda deve poder alterar manualmente a flag
durante a criacao.

---

## 2. Acceptance Criteria

| ID  | Criterio                                                                         | Prioridade |
|-----|----------------------------------------------------------------------------------|------------|
| AC1 | Novos merchants criados tem Funding On Hold = FALSE por default                  | MUST       |
| AC2 | Merchants existentes permanecem sem alteracao                                    | MUST       |
| AC3 | O fluxo de criacao nao mais define Funding On Hold como TRUE por default         | MUST       |
| AC4 | Usuario pode alterar manualmente a flag durante a criacao                        | MUST       |
| AC5 | O fluxo de criacao de merchant continua funcionando normalmente apos o ajuste    | MUST       |

### CT-002 -- Criar merchant sem alterar flag -> persiste como FALSE [AC1, AC2, AC3, AC5]

**Objetivo:** Verificar que ao salvar um novo merchant sem tocar na flag "Funding On Hold", o

**Prioridade:** CRITICO

**Pre-condicoes:**
- Usuario `manager` autenticado em Origination (qa1)

**Passos:**
1. Navegar para `/merchant/new`

   - `legalName`: `QA Funding Hold Legal`

**Asserções:**
- Toast de sucesso visivel

- O merchant acessivel via `/merchant/QA-FOH-{runId}` sem erro

**Validacao de activity log:**
- Tabela candidata: `uown_merchant_activity_log` (schema confirmado no projeto: `merchant_pk`,
  `log_type`, `notes`, `created_by` -- ver `database.helpers.ts` interface `ProgramActivityLogRecord`)
- Query: `SELECT * FROM uown_merchant_activity_log WHERE merchant_pk = $1 ORDER BY pk DESC LIMIT 5`

### CT-005 -- Clone de merchant com FOH=TRUE herda o valor TRUE (comportamento intencional) [AC4]

**Objetivo:** Verificar que ao clonar um merchant com `hold_funding=true`, o formulario do novo
merchant pre-preenche "Funding On Hold" como `true`, permitindo que o usuario veja e ajuste antes
de salvar.

**Prioridade:** MEDIO

**Status:** COMPORTAMENTO CONFIRMADO e INTENCIONAL (confirmado pelo usuario em 2026-06-02).

do AC1/AC3 aplica-se apenas ao formulario vazio (sem clone), nao ao fluxo de clone.

**Passos:**
1. Navegar para `/merchant/new`
2. Clicar "Clone" > buscar "Sunshine" > selecionar "Sunshine Furniture DO NOT USE - Hayward"
   (ref=`00000`, pk=3020, `hold_funding=true` no DB)

4. Verificar `input[name="holdFunding"]`

**Asserções:**

- O usuario pode ver o valor e alterar antes de salvar (campo editavel)

**Validacao de activity log:** N/A -- sem save
