**BUG-01: paymentDetailsList returns invalid 13m links when merchant has no 16m program**

- **Scenario:** Lead DECLINED in 13m program + `isEligibleForExtraInfo=true` + merchant without 16m program  
- **Current behavior:** Backend returns `paymentDetailsList` with 13-month plans (WK13, BW13) whose `redirectUrl` leads to an “invalid link” page  
- **Expected behavior:** `paymentDetailsList` should be empty (empty array), since no alternative program (16m) is available  
- **Impact:** Customer receives broken payment links, leading to frustration and potential conversion loss  
- **Evidence:** URL `https://secure-stg.uownleasing.com/wn6LmKrC/complete?planId=WK13` returns an empty page with `missing-fields` endpoint and no useful data. Frontend renders “invalid link”  
- **Environment:** stg  
- **Severity:** Medium — impacts UX in second look scenario when merchant only has 13m
