## Network Interception Contract

### URL patterns to capture

| Pattern | Why |
|---------|-----|
| `*neuroid.com*` / `*neuro-id.com*` / `*nid.com*` / `*neuro*` (substring) | Frontend SDK bundle + API calls |
| Query/body contains `items340` or `depth355` (siteids) | Deterministic match when domain is masked or routed via CDN |
| `/uown/los/sendApplication` (response only) | Backend response path that processes NeuroID result |
| `/uown/los/submitApplication` (response only) | Same |
| `/uown/los/neuroid*` (any path) | Any direct backend-to-NeuroID proxy |

### Capture structure

Each intercepted event recorded as:

```typescript
interface CapturedNetwork {
  direction: 'request' | 'response';
  url: string;
  method: string;
  status?: number;               // response only
  headersSubset: Record<string, string>;   // content-type, x-uown-*, x-siteid
  requestBody?: string | object; // if small + textual (filter >50KB)
  responseBody?: string | object;
  timestamp: string;             // ISO
  matchedBy: 'domain' | 'siteid' | 'backend-endpoint';
}
```

### New helper to create

**`src/helpers/network-intercept.helper.ts`** — exports:

```typescript
export function attachNeuroIdListeners(
  page: Page,
  buffer: CapturedNetwork[],
): void;
// Installs page.on('request') + page.on('response').
// For each event: apply filter below. On match, push to buffer.

export async function dumpCaptured(
  buffer: CapturedNetwork[],
  label: string,
): Promise<string>;
// Serializes buffer, truncates oversized bodies (>50KB → '...TRUNCATED'),
// attaches via test.info().attach(`network-neuroid-${label}.json`, { body: ..., contentType: 'application/json' }).
// Returns JSON string for logging.

export function classifyNeuroIdOutcome(buffer: CapturedNetwork[]): {
  statusSeen: string[];    // e.g. ['PROFILE_NOT_FOUND']
  backendDecision: 'APPROVE' | 'DECLINE' | 'UNKNOWN';
};
```

### Filter + dedup strategy

1. **Match filter (keep if any):** URL regex `/neuro[-_]?id|nid\.com/i`, OR URL/body contains `items340|depth355`, OR URL contains `/uown/los/(sendApplication|submitApplication|neuroid)`.
2. **Dedup:** by `${direction}|${method}|${url}|${timestamp.slice(0,19)}` (second-precision) — prevents request+response doubles when both listeners fire for overlapping events.
3. **Body size guard:** `if (body.length > 50_000) body = body.slice(0,50_000) + '…TRUNCATED'`.
4. **PII scrubbing:** before attach, mask SSN (`\d{3}-?\d{2}-?\d{4}`), full card numbers (`\d{13,19}`), CVV pattern. Security rule compliance.

## Edge Cases

1. **Listeners attached after redirect** — if `ctx.contractUrl` triggers a redirect to NeuroID domain BEFORE listener install, events are lost. Mitigation: attach in CT-04a BEFORE `page.goto`.
2. **Siteid not in URL nor body (encrypted)** — fallback to domain match. If both fail, `captured` may be empty for qa2 if NeuroID SDK is masked behind CDN. Soft-assert `captured.length >= 0` and log a warning instead of failing — the DB validation is the authoritative signal.
3. **DB audit log table name unknown** — first impl step: query `information_schema.tables WHERE table_name ILIKE '%lead%log%'` against qa2; fall back to reading Java entity.
4. **E-sign iframe doesn't load in qa2** — existing `completeESign()` handles timeout; may require longer sleep in CT-07b. 240s budget already allocated.
5. **CT-09 state post-submit unclear** — dev comment in work item says "only submit visible"; if actual behavior is different, test fails in assertion and is treated as discovery (document in report under "Observations").
6. **Saslow's merchant not yet in qa2 data** — if `buildTestData('SaslowsJewelers')` throws, test MUST fail early in Phase 0 with a clear message; do not silently skip.
7. **PII leak in captured artifact** — scrubber regex must be applied BEFORE `test.info().attach`. Security rule (security.md §2).
8. **Parallel runs sharing listeners** — listener buffer `captured` is per-test (local `const`), so parallel workers don't conflict. Confirmed safe.
