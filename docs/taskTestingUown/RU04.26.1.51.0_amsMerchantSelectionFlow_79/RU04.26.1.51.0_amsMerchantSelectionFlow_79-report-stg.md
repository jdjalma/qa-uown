## Descrição

O cartão "Edit User Merchants" na página de detalhes do usuário (`/users/[username]`) no portal AMS foi substituído por um novo componente. O novo fluxo inclui:

- **Badges de comerciantes** em modo read-only com nome e código do merchant
- **Botão pencil (edit)** para entrar no modo de edição
- **"Manage merchants"** — botão que abre a interface de gerenciamento de comerciantes (modal)
- **"Delete All"** — botão que remove todos os comerciantes atribuídos
- **SAVE / CANCEL** — botões para confirmar ou cancelar a edição
- **Persistência via API** — alterações são salvas via `PUT /user/{username}` e confirmadas via `GET /user/{username}`

## Cenários

### CT-01

**Objetivo:** Verificar que o modo read-only exibe corretamente os badges de comerciantes atribuídos ao usuário.

**O que é verificado:** Ao acessar a página de detalhes do usuário `test6` e expandir o cartão "Edit User Merchants", o sistema exibe badges read-only. Como `test6.merchantCodes="*"` (wildcard), o componente renderiza badges genéricos com label `"All"`.

Exemplos:
| Badge # | Conteúdo (stg / test6) |
|---------|------------------------|
| 1 | All |
| 2 | All |

> Console: `[CT-01] Merchant badges (2): All | All`

#### Como verificar manualmente

1. Login no portal AMS em `https://ams-website-stg.uownleasing.com/` com credenciais admin
2. Navegar para `https://ams-website-stg.uownleasing.com/users/test6`
3. Localizar o cartão "Edit User Merchants" e expandir se colapsado
4. Verificar que os badges de comerciantes estão visíveis (≥ 2 badges; para wildcard exibem "All")
5. Confirmar via API: `GET https://ams-stg.uownleasing.com/user/test6` (headers `sub-system: ams-auth`, `username: <admin>`) — `merchantCodes` deve ser `*`

**PASSOU**
