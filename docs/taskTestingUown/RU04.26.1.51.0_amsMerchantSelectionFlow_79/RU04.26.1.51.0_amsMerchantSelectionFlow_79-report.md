## Descrição

O cartão "Edit User Merchants" na página de detalhes do usuário (`/users/[username]`) no portal AMS foi completamente substituído por um novo componente. O novo fluxo inclui:

- **Badges de comerciantes** em modo read-only com nome e código do merchant
- **Botão pencil (edit)** para entrar no modo de edição
- **"Manage merchants"** — botão que abre a interface de gerenciamento de comerciantes (modal)
- **"Delete All"** — botão que remove todos os comerciantes atribuídos
- **SAVE / CANCEL** — botões para confirmar ou cancelar a edição
- **Persistência via API** — alterações são salvas via `PUT /user/{username}` e confirmadas via `GET /user/{username}`

A implementação real utiliza badges verdes para visualização read-only, com controles de edição (Manage merchants, Delete All) que aparecem ao clicar no pencil. A persistência é verificada tanto via UI quanto via API.

### CT-05

**Objetivo:** Verificar a persistência de dados via API com operação de round-trip (GET, PUT, GET).

**O que é verificado:** O sistema persiste corretamente as alterações de `merchantCodes` quando atualizadas via API. Um ciclo completo de leitura, atualização (adicionando um código de teste) e releitura confirma que o backend do AMS aceita e armazena os dados corretamente, garantindo que a camada de persistência funciona independentemente da UI.

Exemplos:
| Operação | merchantCodes |
|----------|---------------|
| GET inicial | `KS16775,KS16008,KS13418,KS16928,KS15500,KS16059,214` |
| PUT (adicionado código de teste) | `KS16775,KS16008,KS13418,KS16928,KS15500,KS16059,214,TEST_API_1776116778480` |
| GET verificação | Contém `TEST_API_1776116778480` |
| PUT restauração | `KS16775,KS16008,KS13418,KS16928,KS15500,KS16059,214` |

#### Como verificar manualmente

1. Obter o estado atual do usuário via API:
   ```
   GET https://ams-qa2.uownleasing.com/user/test.tester
   Headers: sub-system: ams-auth, username: jmendes.gow
   ```
   Anotar o valor de `merchantCodes`

   ```
   PUT https://ams-qa2.uownleasing.com/user/test.tester
   ```
