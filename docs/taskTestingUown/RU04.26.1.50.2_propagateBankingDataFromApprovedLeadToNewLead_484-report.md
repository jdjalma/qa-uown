## Descrição

Nova step `BankDataStep` adicionada ao pipeline de `sendApplication` (após `underwritingCheck`). Quando um cliente retornante aplica SEM banking data, e seu lead anterior aprovado tinha banking data (source=SEND_APP) e era elegível para termos de 16 meses, os dados bancários são automaticamente propagados para o novo lead em `uown_los_bank_account`.

**Condições para propagação (todas devem ser verdadeiras):**

1. Config flag `saveBankDataFromPreviousLead` = true (default)
2. UW status = APPROVED
3. `eligibleTerms` contém "16" (merchant Kornerstone/16m)
4. `previousUW` não está em branco (cliente retornante)
5. Request NÃO inclui banking data (`mainBankAccountNumber` ausente)
6. Lead anterior tem bank account com source = SEND_APP

## Bugs de Aplicação Encontrados

### BUG-01: BankDataStep não propaga banking data apesar de todas as condições atendidas

| Campo | Valor |
|-------|-------|
| **Severidade** | Alta |
| **Ambiente afetado** | qa1 |
| **Reprodutível** | Sim (100%) |

**Descrição:** O `BankDataStep` não cria registro em `uown_los_bank_account` para o novo lead.

**Evidência qa1:**

Lead setup (CT-00, leadPk=11161) retornou `eligible_terms = '13'` ao invés de `'16'`, apesar de KS3015 ter programas 16m ativos (`KW-16-2` pk=208, `KWC-2.3` pk=215) com FL nos states. Lead 11162 (CT-01) não recebeu registro em `uown_los_uwdata` nem em `uown_los_bank_account`.

```sql
-- Lead setup: eligible_terms='13' (esperado: '16')
SELECT lead_pk, previous_uw, eligible_terms, uw_status
FROM uown_los_uwdata WHERE lead_pk = 11161;
-- Resultado: lead_pk=11161, previous_uw=null, eligible_terms='13', uw_status='APPROVED'

-- Lead setup tem SEND_APP
SELECT account_number, routing_number, source
FROM uown_los_bank_account WHERE lead_pk = 11161 AND is_deleted = false;
-- Resultado: pk=4995, account_number='111000025', routing_number='021000021', source='SEND_APP'

-- Novo lead NÃO recebeu propagação
SELECT COUNT(*) FROM uown_los_bank_account WHERE lead_pk = 11162;
-- Resultado: 0 — BankDataStep não criou registro

-- Novo lead nem tem uwdata
SELECT COUNT(*) FROM uown_los_uwdata WHERE lead_pk = 11162;
-- Resultado: 0
```

**Hipótese:** O `BankDataStep` pode não estar deployado em qa1, ou a flag de configuração `saveBankDataFromPreviousLead` está desabilitada. Requer confirmação do time de desenvolvimento.

**Impacto na tela:** A tela do contrato (`/complete?planId=...`) solicita dados bancários novamente na 2ª aplicação, quando deveria vir pré-preenchida.
