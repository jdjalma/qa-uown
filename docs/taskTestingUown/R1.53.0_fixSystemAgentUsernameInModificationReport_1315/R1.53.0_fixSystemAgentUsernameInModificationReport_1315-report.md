## Descrição

`uown_lead_modifications.agent_username` registrava `"SYSTEM"` para transições de status (`LEAD_STATUS_CHANGE`) disparadas por agentes humanos no portal Origination. Causa raiz: `ChangeLeadStatusService.changeLeadStatus()` lia o nome do agente de `ThreadAttributes.getUsername()` DEPOIS de um webhook de saída que corrompia o `ThreadLocal`, retornando vazio → default `"SYSTEM"`. O fix captura `agentName` no início do método, antes de qualquer chamada que possa corromper o `ThreadLocal`.

Mecanismo crítico: o portal SPA envia o header HTTP `username`. Chamadas API diretas sem esse header reproduzem o artefato `SYSTEM` — por isso a ação de status change é exercida **obrigatoriamente via UI** (regra #14), autenticada como `jmendes.gow`.
