## Source-verified clarifications (2026-05-20 final)

### Finding #1 — `agent` column on `uown_sv_inbound_api_log` / `uown_los_inbound_api_log` is ALWAYS NULL

**Source-verified facts (file:line):**

- `../svc-common/src/main/java/com/uownleasing/svc/common/db/entity/SvInboundApiLog.java` — entity has no `@EntityListeners`, no `@PrePersist` that calls `setAgent()`.
- `../common/src/main/java/com/uownleasing/common/db/entity/superentity/SuperEntity.java` — `@PrePersist onCreate()` only sets timestamps; `setAgent()` method exists but is never invoked from a listener.
- `../svc/src/main/java/com/uownleasing/svc/superentity/SuperEntity.java` — same pattern, no auto-agent population.
- `../svc/src/main/java/com/uownleasing/svc/logging/AspectInboundApiLog.java` — never calls `entity.setAgent(...)` on `SvInboundApiLog` or `LosInboundApiLog` instances before `saveAndFlush`.

**Conclusion:** the `TMS-` prefix lives in `ThreadAttributes.username` (in-memory thread-local). The audit log row (`agent` column on `uown_sv_inbound_api_log` and `uown_los_inbound_api_log`) is **ALWAYS NULL**. The prefix is consumed downstream by services that EXPLICITLY call `setAgent(ThreadAttributes.getUsername())` on OTHER entities — e.g. `FrequencyMods` (`:21`), `MerchantLogService` (`:64`, `:93`), and `LoggingService.createActivityLog` via `setCreatedBy(finalUsername)` (`:104-117`).

**Implication for testing:** the `TMS-` prefix MUST be asserted on the downstream business entity (`sv_activity_log.created_by` for activity-log paths), NOT on `sv_inbound_api_log.agent`. The audit row only proves the AOP fired; the prefix logic is observable one hop downstream. CT-10 below is reformulated accordingly. Resolves Q-A3.

### One-line problem statement

The AOP advice in `svc` (`AspectInboundApiLog`) that mirrors every inbound TMS and LOS HTTP request into the `uown_sv_inbound_api_log` / `uown_los_inbound_api_log` audit tables was missing pointcut coverage for (a) the modern RESTful TMS controllers under `com.uownleasing.svc.rest.tms.*` (`TmsAccountController` / `TmsPaymentController` / `TmsDueDateController` — base `/uown/tms/v1/...`), (b) the legacy `TmsController` FQCN match for the `TMS-` username prefix, and (c) `LosExternalMerchantController` for LOS-side logging. Fix extends both `@Around` pointcuts and corrects the FQCN comparison.

### One-line scope statement

Pure backend audit-trail / observability bug. **No UI affordance**, no customer-facing behavior change, no business-state mutation. Verifiable only via HTTP call + DB read. Implementation must therefore exercise the AOP **as a black box at the HTTP boundary**, not via internal mocks.
