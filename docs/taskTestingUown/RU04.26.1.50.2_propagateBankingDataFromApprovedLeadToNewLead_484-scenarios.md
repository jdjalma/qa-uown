## SQL Queries

### Setup: Find existing approved lead with banking data and 16m eligibility

```sql
-- Find a lead that:
-- 1. Has been approved (lead_status beyond UW_APPROVED)
-- 2. Has banking data in uown_los_bank_account with source = 'SEND_APP'
-- 3. Is associated with a KS3015 merchant (eligibleTerms contains "16")
-- 4. Has previous UW data (previousUW condition met by having uown_los_uw_info)

SELECT l.pk AS lead_pk, l.customer_pk, l.lead_status, l.merchant_number
FROM uown_los_lead l
JOIN uown_los_bank_account ba ON ba.lead_pk = l.pk AND ba.is_deleted = false AND ba.source = 'SEND_APP'
JOIN uown_los_uwdata uw ON uw.lead_pk = l.pk
WHERE l.lead_status IN ('UW_APPROVED', 'CC_AUTH_PASSED', 'CONTRACT_CREATED', 'SIGNED', 'SETTLED', 'FUNDING', 'FUNDED')
  AND l.merchant_number = 'KS3015'
  AND uw.eligible_terms LIKE '%16%'
ORDER BY l.pk DESC
LIMIT 1;
```

### Setup: Extract SSN from customer

```sql
SELECT ssn FROM uown_los_customer WHERE pk = $1;
-- $1 = customer_pk from the lead above
```

### Setup: Get previous banking data for comparison

## Edge Cases

| # | Edge Case | Handling |
|---|-----------|----------|
| 1 | **No eligible lead found in qa2** | Setup step should fail fast with descriptive error: `expect(existingLeadPk, 'No approved lead with banking data (source=SEND_APP) and 16m eligibility found in qa2. Pre-seed required.').toBeTruthy()` |
| 2 | **No approved lead WITHOUT banking data in qa2** | CT-04 setup may fail. Provide fallback: skip CT-04 with `test.skip(!noBankLeadPk, 'No approved lead without banking data found in qa2')` |
| 3 | **SSN concurrency lock** | `sendApplication` uses `ConcurrentHashMap` by SSN. If another test is using the same SSN simultaneously, returns `"Application already in progress"`. Mitigated by: (a) sequential mode, (b) each CT uses the same SSN but serially |
| 4 | **Previous UW reuse** | The `previousUW != blank` condition requires the UW engine to detect the customer as a returning applicant and reuse previous UW data. If the customer's previous approval expired, UW runs fresh and `previousUW` may be blank. Choose leads with recent approval (within `numDaysApprovalExp`) |
| 5 | **Banking data already deleted (is_deleted=true)** | Query explicitly filters `is_deleted = false`. If all bank account records for the previous lead are soft-deleted, propagation won't happen. Setup query already filters for this |
| 6 | **`buildSendApplicationBody` adds default banking data** | The builder does NOT add `mainBankAccountNumber`/`mainBankRoutingNumber` by default (checked in source). But verify the body doesn't contain these fields before calling sendApplication in CT-01/CT-03/CT-05 |
| 7 | **Response `accountNumber` is UUID, not lead PK** | The `sendApplication` response returns `accountNumber` which is the lead UUID. Must use `db.getLeadPkByUuid()` to get the numeric `pk` for DB queries. `node-pg` returns bigint as string |
| 8 | **`eligible_terms` column format** | May be stored as comma-separated string (e.g., `"13,16"`) or JSON. The `LIKE '%16%'` query handles both formats |
