### BUG-01 — `{{nextPaymentDueAmount}}` ausente do variables map no dispatch GowSign (OH 16m)

- **Status:** [CONFIRMADO]
- **Severidade:** S2 (contrato legal renderiza taxa de pagamento em branco — exposição compliance) / Prioridade P1
- **Descrição:** ao dispatchar o documento GowSign do template OH `OH_2025_SAC_16_MONTHS`, o backend (`DocumentDispatchService`) reporta o token `nextPaymentDueAmount` como **missing** no variables map. Como o token não tem valor para substituir, os slots Item 3 ("Regular MONTHLY lease rate is $***") e Item 4a ("16 payments of $***") renderizam **em branco** — exatamente o modo de falha "blank, não leaked" que AC4 protege. O valor existe e está correto no invoice (oracle MONTHLY `nextPaymentDueAmount=95.16`), mas não é propagado para o dispatch do documento.
- **Como Reproduzir:**
  1. Criar lead OH 16m fresh no `OL90205-0079_clone` (sticky SSN, Bucyrus OH) → UW_APPROVED.
  2. `sendInvoice` MONTHLY $662 → confirmar `firstPaymentWithFeesNoTax = 95.16` no `paymentDetailsList`.
  3. Submeter CC / abrir contrato (gera o esign_document GOWSIGN).
  4. `SELECT notes FROM uown_los_lead_notes WHERE lead_pk = :leadPk AND notes ILIKE '%[DocumentDispatchService][GowSign]%variables map missing%'`.
- **Evidência (fonte primária):**
  - Lead 16665: `[DocumentDispatchService][GowSign] leadPk=16665 variables map missing 1 simple template token(s): [nextPaymentDueAmount]` (note pk=148496) `[db-observation:lead_pk=16665]`
  - Lead 16667: `[DocumentDispatchService][GowSign] leadPk=16667 variables map missing 1 simple template token(s): [nextPaymentDueAmount]` (note pk=148620) `[db-observation:lead_pk=16667]`
  - Oracle prova o valor existe: `firstPaymentWithFeesNoTax=95.16` (MONTHLY) `[api-response:sendInvoice,200]`
  - **Escopo CONFIRMADO: UNIVERSAL nas 3 frequências do template OH 16m** (não MONTHLY-specific). Run 5 (um lead por frequência) — WEEKLY (16678), BI_WEEKLY (16679), MONTHLY (16681): nos 3, o Item 3 renderiza em branco E o dispatch note registra `variables map missing [nextPaymentDueAmount]`. `[test-execution:b3kx6n789]` `[db-observation:lead_pk IN (16678,16679,16681)]`. As demais variáveis (costPrice $711, contract $1681.99) renderizam corretas nas 3 → defeito isolado nesse token.
  - É template-specific (OH 16m), não global: o padrão aparece em ~9 leads/11 rows desse template, NÃO nos 358 dispatches GowSign. (Nota: `cashPrice`-missing em 639 rows = defeito SEPARADO, fora do escopo.)
- **Cenário que detectou:** CT-01 (deliverable do teste) — confirmado pelo dispatch log mesmo com o render visual ainda bloqueado por F-002.
- **Causa provável:** binding de variável no dispatch do template OH 16m não inclui `nextPaymentDueAmount` no map passado ao GowSign. `[db-observation:DocumentDispatchService log]` — cabe investigação do time de dev no `DocumentDispatchService` / mapeamento de tokens do template OH (não recomendo fix de código em evidência única; sintoma + evidência documentados).
