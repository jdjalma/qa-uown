## Description

Store and display approved lease term months for a lead. The system persists underwriting-approved terms and merchant-available terms in a new DB table (`uown_lead_approval_terms`) and shows the approved terms in the Merchant Info section of the Origination customer page.

**Key changes:**
- New DB table `uown_lead_approval_terms` (insert-only history)
- New `termsStep` in origination pipeline (after underwriting, no-invoice leads only)
- `CalculatorStep` extended to also write terms (invoice flow)
- `GET /uown/los/getMerchantInfo/{leadPk}` returns new field `approvedTermMonthsDisplay`
- Origination UI: read-only "Approved term months" row in Merchant Info panel

## Key Findings

| Observation | Detail |
|-------------|--------|
| **SSN format** | API aceita SSN sem traços (`888888888`, não `888-88-8888`) |
| **approved_terms (lista)** | SSN 888888888 → DB armazena `"13,16"` (comma-separated); API exibe `"13 months, 16 months"` |
| **approved_terms (único)** | SSN 888880916 → DB armazena `"16"`; API exibe `"16 months"` |
| **STG DB columns** | STG tem colunas adicionais: `pk`, `row_updated_timestamp`, `tenant_id` (além das 5 requeridas) |
| **merchantNumber override** | Catalog number (`GOW-0003_clone_fer_ks`) substituído por `KS1337` (qa2) e `KS1337-001` (stg) via override no testData |
| **merchantPassword override** | STG usa password diferente do catalog — suportado via campo `merchantPassword` no testData |
| **getApprovedTermMonths()** | Usa `evaluate()` no DOM — encontra elemento `<label>Approved term months</label>` e retorna o `nextElementSibling.textContent` |
| **XPath fix** | Selector usa `normalize-space(.)` (não `text()`) para capturar elementos com texto em filho aninhado |
| **Video highlight** | Campo destacado com `outline: 3px solid #ff6b00` + `backgroundColor: #fff3e0` + scroll + 3s delay para visualização |
