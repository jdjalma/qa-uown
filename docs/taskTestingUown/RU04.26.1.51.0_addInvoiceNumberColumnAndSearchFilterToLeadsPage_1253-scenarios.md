## Análise Técnica

### Endpoint
- `POST /uown/los/getLeadsByCriteria`

### Novo parâmetro de filtro
- `invoiceNumber` (string) — busca LIKE case-insensitive em `uown_los_invoice.merchant_invoice_number`

### Novo campo na resposta
- `invoiceNumber` (string | null) — via LEFT JOIN `uown_los_invoice` no `lead_pk`

### Lógica SQL
```sql
LEFT JOIN uown_los_invoice invoice ON invoice.lead_pk = lead.pk
    AND (:invoiceNumber IS NULL OR LOWER(invoice.merchant_invoice_number) LIKE LOWER(:invoiceNumber))
WHERE ...
    AND (:invoiceNumber IS NULL OR invoice.pk IS NOT NULL)
```
- Sem filtro → todos os leads retornam; coluna vazia para leads sem invoice
- Com filtro → apenas leads com invoice correspondente são retornados

### Frontend
- Filtro: text input, placeholder `'Search by Invoice Number'`, label `'Invoice Number'`
- Coluna: `'Invoice Number'`, chave `invoiceNumber`, posicionada após `'Term Month'`
- DB source: `uown_los_invoice.merchant_invoice_number`

### Dados de teste
- **Lead COM invoice**: buscar via DB — `SELECT l.pk, i.merchant_invoice_number FROM uown_los_lead l JOIN uown_los_invoice i ON i.lead_pk = l.pk WHERE i.merchant_invoice_number IS NOT NULL ORDER BY l.pk DESC LIMIT 1`
- **Lead SEM invoice**: buscar via DB — `SELECT l.pk FROM uown_los_lead l LEFT JOIN uown_los_invoice i ON i.lead_pk = l.pk WHERE i.lead_pk IS NULL AND l.lead_status NOT IN ('UW_DENIED','ABANDONED') ORDER BY l.pk DESC LIMIT 1`

**Quero** ver o número da fatura na tabela de Leads,
**Para** identificar rapidamente leads associados a faturas específicas sem abrir cada lead individualmente.

### Critérios de Aceite
