## Descrição

Implementação da página de exibição de Payment Arrangements no portal Servicing. A funcionalidade permite visualizar a lista de arrangements de um account, expandir cada arrangement para ver os pagamentos individuais (CC/ACH), e acompanhar o ciclo de vida completo: criação, processamento de parcelas e conclusão (SUCCESS). Inclui validação de mascaramento de cartão, estrutura paginada da API e persistência dos dados após refresh.

## Cenários

### Cenário: Cenário 1 — CT-01

**O que é feito:** Chama `POST /uown/svc/makeCreditCardPayments` com `accountPk=4368`, `arrangementType` omitido (default NORMAL), cartao `VALID_TEST_CARDS[0]` (last4=0055), 3 parcelas de $100 cada com `postingDate` = hoje, +7 dias, +14 dias. Formato de data ISO (`calculateDateISO`).

**O que acontece:** API cria o arrangement na tabela `uown_sv_payment_arrangement` com `arrangement_type=NORMAL`, `payment_type=CC`, `is_active=true`. A 1a parcela (hoje) e processada sincronamente para APPROVED. As 2 parcelas futuras ficam PENDING em `uown_sv_credit_card_transaction`.

**O que é verificado:**
- HTTP 200 na resposta da API
- DB: `uown_sv_payment_arrangement` — `arrangement_type=NORMAL`, `payment_type=CC`, `is_active=true`, `arrangementPk=204`
- DB: `uown_sv_credit_card_transaction` — 3 registros vinculados por `payment_arrangement_pk=204`: 1 com `status=APPROVED`, 2 com `status=PENDING`

Exemplos:
| Campo | Valor |
|-------|-------|
| accountPk | 4368 |
| arrangementPk | 204 |
| arrangement_type | NORMAL |
| payment_type | CC |
| is_active | true |
| CC txns APPROVED | 1 |
| CC txns PENDING | 2 |

#### Como verificar manualmente

1. Executar no Postman: `POST {svcUrl}/uown/svc/makeCreditCardPayments` com body contendo `accountPk=4368`, 3 installments ($100 cada), cartao de teste valido
2. Verificar resposta HTTP 200
3. Consultar DB:
   ```sql
   SELECT pk, arrangement_type, payment_type, status, is_active
   FROM uown_sv_payment_arrangement
   WHERE account_pk = 4368
   ORDER BY pk DESC LIMIT 1;
   ```
4. Consultar transacoes CC:
   ```sql
   SELECT pk, status, posting_date, amount
   FROM uown_sv_credit_card_transaction
   WHERE payment_arrangement_pk = 204
   ORDER BY posting_date ASC;
   ```
5. Confirmar: 1 APPROVED (posting_date = hoje) + 2 PENDING (posting_date futuro)

**PASSOU**
