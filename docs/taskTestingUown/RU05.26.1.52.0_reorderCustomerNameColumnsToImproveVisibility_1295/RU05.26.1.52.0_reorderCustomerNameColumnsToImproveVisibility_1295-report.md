### Improvements identified (do not block this task's deploy)

**1. `Customer Name` sort in Leads does not order alphabetically**
Clicking the `Customer Name ▲` header in Leads sets the active sort indicator, but the rows are not reordered lexicographically. Sample observed in qa1:
`TestFNlej TestLNlej`, `Fernando Martins`, `TestFNw TestLNw`, `Jane Doe`, `Lucas Elias`, `TestFNe TestLNe`, …
The header reacts visually but the data sort is not applied (or applies a non-name criterion).
→ Suggestion: open a separate ticket with Marcus/Yuri to confirm whether lexicographic sort on `Customer Name` is expected; if so, fix the handler.

**2. Lowercase / case-sensitivity of header labels**
Headers in the DOM render as `First Name` / `Last Name` (Title Case), not the all-caps `CUSTOMER FIRST NAME` referenced in the ticket description. Behavior matches the visible labels in the UI today — no impact on the fix, just a doc note for future tickets that reference these columns.
