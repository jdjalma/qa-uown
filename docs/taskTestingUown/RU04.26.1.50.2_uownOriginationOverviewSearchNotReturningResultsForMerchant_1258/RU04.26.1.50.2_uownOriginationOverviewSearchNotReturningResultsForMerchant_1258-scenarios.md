# Test Scenarios — #1258 Search Not Returning Results for Merchant with Apostrophe

## Task Info
- **GitLab:** https://gitlab.com/uown/frontend/origination/-/work_items/1258
- **MR:** https://gitlab.com/uown/backend/svc/-/merge_requests/1347
- **Milestone:** RU04.26.1.50.2 (Hotfix)
- **Tipo:** Bug fix — double-escaping de apóstrofos na busca de merchants

## Root Cause
`LeadFilters.sanitize()` e `ApplicationService` faziam `.replace("'", "''")` manual nos campos `merchants`, `locations`, `search` e `merchantSupport`. Como o `NamedParameterJdbcTemplate` do Spring já parametriza automaticamente, o replace manual gerava double-escaping (`Daniel''s Jewelers`), quebrando o match no SQL.

## Fix (MR #1347)
Removido `.replace("'", "''")` de:
1. `LeadFilters.sanitize()` — campos `merchants` e `locations`
2. `ApplicationService.getLeadsInDateRange()` — campo `merchantSupport`
3. `LeadFilters.sanitize()` — campo `search`

## Escopo Ampliado — Cross-Page Regression
Além do Overview (endpoint afetado pelo fix), testar TODAS as páginas com filtro de Merchant/Location para garantir que apóstrofos funcionam corretamente em todo o sistema.

### Mapa de Páginas × Endpoints × Parameterização

| # | Página | URL | Endpoint | Param Type | Merchant Select | Page Object |
|---|--------|-----|----------|------------|-----------------|-------------|
| 1 | **Overview** | `/overview` | `getLeadsInDateRange` | NamedParam (FIXED) | Multi-select | `OverviewPage` |
| 2 | **Open to Buy** | `/openToBuy` | `getOpenToBuyCustomersList` | NamedParam | Multi-select | `OpenToBuyPage` |
| 3 | **New Application** | `/newApplication` | `getSendApplicationRequestsByCriteria` | NamedParam | Single-select | Nenhum (criar) |
| 4 | **Merchant Mod History** | `/merchantModificationHistory` | `getMerchantDataChangeResults` | NamedParam | Single-select | Nenhum (criar) |
| 5 | **Modification Report** | `/modificationReport` | `getModifiedLeads` | NamedParam | Single-select | Nenhum (criar) |
| 6 | **Funding** | `/funding` | `getLeadsForFundingQueue` | **String interpolation** | Single-select | `FundingPage` (sem filtro merchant) |
| 7 | **Leads** | `/leads` | `getLeadsByCriteria` | NamedParam | Single-select | `LeadsPage` |
| 8 | **Servicing Search** | `svc-website/search` | `getAccountsByCriteria` | NamedParam | Single-select | `ServicingSearchPage` |

> **Nota Funding:** `LeadFundingService.java` usa `.replaceAll()` com string interpolation (não NamedParam). O `.replace("'","''")` é **correto** neste caso porque os valores são interpolados diretamente no SQL.
