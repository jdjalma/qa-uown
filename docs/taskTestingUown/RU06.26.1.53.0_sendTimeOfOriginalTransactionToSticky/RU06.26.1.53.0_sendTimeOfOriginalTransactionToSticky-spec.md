## US-567-01 — Original-transaction time travels in the Sticky recovery request

**As** o vendor de recovery Sticky (consumidor do payload `StickyRecoveryRequest`),
**I want** receber o horário da transação original (falha) em UTC dentro de cada request de recovery,
**So that** eu consiga reconciliar a tentativa de retry contra o charge original que falhou.

### Persona / journey

Não há persona humana — é uma **integração backend system-to-system** disparada por sweep agendado (`StickyRecoverSweep`), sem affordance de UI no portal (Origination/Servicing/Website/AMS). A "jornada" é: CCT (SALE) DENIED → permanece elegível ~7 dias → sweep seleciona → svc monta e envia `StickyRecoveryRequest` ao Sticky → registra em `uown_sticky_outbound_log`. A verificação QA é data-layer porque o "usuário" (Sticky) só vê o JSON do request. `[skill:user-journey-perspective]` `[svc-source:StickyRecoverSweepService.java:44-71]`

- **Type:** API+DB (cross-cutting sobre output de CT-01/CT-02). **Portal:** N/A.
