## Descricao

MR !1464 ajusta o proxy nginx para strip do prefixo `/iojs/` antes de encaminhar ao upstream FraudForce (iovation). O bundle frontend continua referenciando `/iojs/general5/IOVATION_KEY/static_wdp.js` e `/iojs/.../dyn_wdp.js`; nginx substitui `IOVATION_KEY` server-side e remove o prefixo. Resultado: scripts retornam 200, IGLOO loader inicializa, e `uown_los_lead.iovation_fingerprint_text` recebe valor real (nao-placeholder) em toda aplicacao nova.

O teste implementa 3 camadas de assercao por CT:
- **L1** inspecao de `window.IGLOO.loader` (campos `version`, `subkey`, `uri_hook`, `loaderVer`, `tp_host`)
- **L2** captura/validacao de responses `/iojs/.../wdp.js` (status 200) via `page.on('response')`
- **L3** query em `uown_los_lead.iovation_fingerprint_text` (nao-nulo, nao-placeholder, length > 100)
