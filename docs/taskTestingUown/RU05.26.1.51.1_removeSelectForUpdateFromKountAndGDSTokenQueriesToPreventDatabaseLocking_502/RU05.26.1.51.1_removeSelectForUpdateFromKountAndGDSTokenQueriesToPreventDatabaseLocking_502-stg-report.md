## Mudanças aplicadas no spec antes deste run

**CT-02-alt refatorado.** A versão anterior observava o tick natural do cron (`0 */10 * * * ?`) e validava `uown_scheduled_task.last_trigger_time`. Premissa errada (confirmado por **Davi Artur, 2026-05-07**): o endpoint `/uown/svc/refreshKountAccessTokenSweep` (e o GDS análogo) **não escreve** `last_trigger_time` — só o caminho do scheduler `/uown/svc/triggerScheduledTask/{name}` persiste esse campo.

Refactor:
- Endpoint trocado: `api.scheduledTask.refreshKountAccessTokenSweep()` → `api.scheduledTask.triggerScheduledTask('refreshKountAccessTokenSweep')` (e idem para GDS).
- Sleep de 11min eliminado → 2s de tx-flush margin por sweep.
- `test.setTimeout(900_000)` removido (default de 120s sobra).
- Tag `@nightly` removida; CT-02-alt vira parte do run normal.
- Cobertura ampliada para os dois sweeps (kount + gds) na mesma execução.

Nenhuma outra alteração no spec.
