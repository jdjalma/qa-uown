## Informações da Tarefa

- **Título:** Complete Application — Security: PII data access reduced
- **Milestone:** RU06.26.1.53.0
- **Contexto:** Security fix — `basicCustomerData` E `protectionPlan` (PII) removidos do `localStorage`/`utilityStore` da Complete Application page (origination!1466). PII também removido do submit response (FE model + BE POJO, svc!1454). Gate `MissingRequiredFieldsService` (svc!1454) só popula `basicCustomerData` na complete-application response quando `idCheckProvider==SEON` OU `isOfferInsuranceRequired==true`. SEON (ID verification) e Buddy (purchase protection) devem continuar funcionais.

## Descrição

A página Complete Application antes persistia um blob `basicCustomerData` com PII (nome, SSN, DOB, email, telefone, endereço) no `localStorage`/`utilityStore` do navegador do cliente, e o `protectionPlan` também tinha `@persist`. O fix realocou esses dados para fora do storage do browser (removeu `@persist` de ambos + reset explícito no clear). Os dois consumidores que liam o blob — SEON (verificação de identidade) e Buddy (plano de proteção/seguro) — precisam continuar funcionando, agora alimentados pelo gate condicional. A suíte (endurecida após auditoria contra o diff real) valida o fechamento do leak em contexto anônimo limpo (oráculo PII value-based key-agnostic) e a continuidade funcional de SEON + Buddy (opt-in, opt-out, cross-coverage).

4. Repetir render em 768×1024 e 375×667.
5. DB: `SELECT pk, notes FROM uown_los_lead_notes WHERE lead_pk=$1 ORDER BY pk DESC` — sem nota de "missing data".

**Status:** ⚠️ PARCIAL — oráculos do fix PASSAM; full green bloqueado por flap de env
> Oráculo do gate negativo: **PASS** após recalibração.
