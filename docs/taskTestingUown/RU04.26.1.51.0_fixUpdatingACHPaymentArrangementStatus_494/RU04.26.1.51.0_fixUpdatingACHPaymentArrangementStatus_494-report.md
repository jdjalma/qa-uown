## Descrição

O `PaymentArrangementACHListener` foi refatorado no MR !1368 para resolver classificação incorreta de statuses ACH intermediários. A lógica anterior de `isPending` e `isFailure` deixava alguns statuses em "zona cinza" (ACK_RECEIVED, PICKED_TO_SEND, PENDING_TO_RERUN, CANCELLED, BLOCKED_ACCOUNT), levando a resultados imprevisíveis na avaliação do arrangement.

A refatoração introduziu dois `EnumSet`s explícitos:

- **PENDING** = { PENDING, SENT, ACK_RECEIVED, PICKED_TO_SEND, STATUS_UPDATE_PENDING, PENDING_TO_RERUN }
- **SUCCESS** = { SETTLED, COMPLETED, SETTLED_IN_RERUN }
- **isFailure** = tudo que NÃO está em PENDING nem em SUCCESS (lógica residual)

Prioridade de avaliação: **FAILURE > PENDING > SUCCESS**

Mudanças comportamentais chave:
- ACK_RECEIVED, PICKED_TO_SEND, PENDING_TO_RERUN: antes "zona cinza" (poderia ser FAILED) → agora IN_PROGRESS
- CANCELLED, BLOCKED_ACCOUNT: antes "zona cinza" → agora FAILED (lógica residual)
- Arrangement só avança para SUCCESS quando **todos** os ACH payments estão no set SUCCESS

## Decisões Técnicas

- **Estratégia de simulação DB:** Como o ambiente qa2 não possui Profituity configurado para simulação controlada de statuses, todos os cenários utilizam UPDATE direto no banco + helper `recalculateAchArrangementStatus`, que espelha a lógica Java do MR !1368 com os EnumSets PENDING e SUCCESS.
- **Contas ACTIVE pré-existentes:** Todas as 18 contas foram selecionadas via query no `beforeAll` (contas ACTIVE sem arrangement ativo). Nenhuma conta foi criada.
- **CT-05 destrutivo:** A conta 11096 foi permanentemente transicionada para SETTLED_IN_FULL.
- **Bug #446 corrigido:** `AccountFinancialInfoService.updateRatingLetterAndAutoPay` agora persiste a entidade corretamente. Asserts: `expect(dbRating).toBe('P')` + `expect(logs.length).toBeGreaterThan(0)`.
- **Activity log filtrado por arrangementPk:** O helper `verifyRatingLetterDb` busca com `LIKE '%rating%arrangementPk=XXX%'` em vez de apenas `'%rating%'`, garantindo que o log pertence ao arrangement da execução atual (não de execuções anteriores na mesma conta).
