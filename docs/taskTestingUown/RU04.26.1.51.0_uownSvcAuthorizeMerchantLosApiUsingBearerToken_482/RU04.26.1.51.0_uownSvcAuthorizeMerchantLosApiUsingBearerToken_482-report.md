## Descrição

Merchants integrating with the LOS API should authenticate using a Bearer token in the Authorization header. New merchants will generate an API key and send it as `Authorization: Bearer <token>`. Scope for this release: LOS only. TMS may adopt Bearer for vendors in a follow-up.

Alterações no backend (MR !1332 merged):
- `ApiUserType` enum (`MERCHANT`, `VENDOR`)
- `ApiUserInfo.apiUserType` field
- `RequestFilter` — MERCHANT users must usar Bearer format; VENDOR aceita raw key
- `LosMerchantApiAuthPaths` — define paths que requerem partner auth (`/uown/los/merchant/applications/**`)
- `LosExternalMerchantController` — novo controller v2 para partner endpoints
- Flyway migration `V20260325092624` — coluna `api_user_type` em `uown_api_user`

### Situação Atual

A feature #482 está implementada em **duas camadas independentes**:

| Camada | Responsável | Status | Descrição |
|--------|-------------|--------|-----------|
| **SVC RequestFilter** | Marcus Braga (MR !1332) | Merged | Código aceita `Bearer <token>` para MERCHANT, rejeita raw key |
| **Infraestrutura JWT/Authentik** | Rodrigo Santos | Em andamento | Istio valida JWT via Authentik antes de encaminhar ao SVC |

O Istio intercepta qualquer `Authorization: Bearer *` e valida como JWT via Authentik. Como a hex key retornada por `/auth/authorize` **não é um JWT**, o request é rejeitado antes de chegar ao SVC:

```
Bearer <hex_key> → Istio → 401 "Jwt is not in the form of Header.Payload.Signature"
<hex_key> (raw)  → Istio ignora → SVC RequestFilter → 401 "Invalid API KEY" (MERCHANT exige Bearer)
```

Este comportamento foi **confirmado em 3 ambientes** (qa2, dev2, sandbox) pelo CT-09.

### Impacto nos Testes

Os steps B, C e D da issue dependem de um **JWT real** (não hex key) para funcionar end-to-end. Quando a integração JWT/Authentik estiver completa:
- O `/auth/authorize` deverá retornar um JWT (ou haverá um novo endpoint de autenticação)
- O JWT passará pelo Istio → Authentik → SVC RequestFilter
- Os testes CT-09 (e novos cenários para B, C, D) deverão ser atualizados para usar JWT real

### Ação Necessária

Aguardar alinhamento entre Rodrigo Santos e Marcus Braga sobre o fluxo JWT para então:
1. Atualizar o teste para obter JWT real via novo endpoint/fluxo
2. Implementar cenários positivos para Steps B, C e D com Bearer + JWT
3. Re-executar em todos os ambientes
