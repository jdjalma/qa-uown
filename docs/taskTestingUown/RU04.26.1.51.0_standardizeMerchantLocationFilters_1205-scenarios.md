# Test Scenarios — Task #1205
## Standardize Merchant and Location Filters and Enable Multi-Select

**Milestone:** RU04.26.1.51.0
**Environment:** qa2
**Portal:** Origination
**Type:** E2E (UI)
**Test name:** `RU04.26.1.51.0_standardizeMerchantLocationFilters_1205`

---

## Contexto

A task #1205 criou o componente compartilhado `MerchantLocationFilters` que padroniza os filtros de Merchant e Location em **5 áreas** do módulo Origination. Os Testing Steps foram definidos pelo desenvolvedor (Fernando Martins) no comentário da issue.

### Páginas afetadas e modos

| Página | Merchant Mode | Location Mode | Select All? |
|--------|:---:|:---:|:---:|
| Overview | multi | multi | ✅ |
| Open to Buy | multi | multi | ✅ |
| Leads (Search) | single | single | ❌ |
| New Application | single | single | ❌ |
| Customers → Sales Rep panel | single | single | ❌ |

### Comportamentos chave do componente

**Noções básicas (aplicam-se em todos os lugares):**
- Dropdowns são pesquisáveis (filtra opções ao digitar)
- Páginas single-select: apenas 1 merchant e/ou 1 location
- Páginas multi-select: múltiplos itens + "Select All" usa sentinel `{value: 'all'}` (não "todos selecionados")
- Clear: limpar um campo remove o filtro sem deixar listas em branco

**Select All — exclusividade mútua:**
- Selecionar "Select All" → campo vira somente Select All
- Com Select All ativo, selecionar item específico → campo vira somente aquele item

**Cobertura existente:** Nenhuma — nenhum teste cobre filtros de Merchant/Location nessas páginas.

## User Stories

### US-OR-01: Multi-select Merchant/Location com Select All (Overview e Open to Buy)
**Como** admin do Origination, **Quero** selecionar múltiplos merchants e locations nos filtros (inclusive "Select All"), **Para** visualizar leads de combinações específicas ou de todos os dados.

**Critérios de Aceite:**
- [ ] Selecionar "Select All" em Merchant → campo fica apenas Select All; Location permanece utilizável
- [ ] Com Merchant = Select All, selecionar merchant específico → campo muda para só aquele merchant (exclusividade mútua)
- [ ] Selecionar locations específicas sem merchant → merchants correspondentes são auto-adicionados
- [ ] Selecionar "Select All" em Location → campo fica apenas Select All
- [ ] Com Location = Select All, selecionar location específica → campo muda para só aquela location (exclusividade mútua)
- [ ] Selecionar merchants específicos → Location dropdown restringe-se à união das locations desses merchants

### US-OR-02: Single-select Merchant/Location pesquisável (Leads, New Application)
**Como** usuário do Origination, **Quero** filtrar por merchant e location (single-select pesquisável), **Para** encontrar leads/applications de um merchant específico.

**Critérios de Aceite:**
- [ ] Digitar no dropdown de Merchant filtra as opções em tempo real
- [ ] Selecionar merchant → Location options são restritas às locations desse merchant
- [ ] Selecionar location (sem merchant) → Merchant auto-preenche com o primeiro merchant correspondente
- [ ] Limpar merchant → Location é limpo e options resetadas

### US-OR-03: Sales Rep panel — Editar Merchant/Location do lead
**Como** admin do Origination, **Quero** editar o Merchant e Location de um lead via Sales Rep panel, **Para** corrigir atribuições de merchant quando necessário.

**Critérios de Aceite:**
- [ ] Disponível apenas quando status = UW_APPROVED ou CONTRACT_CREATED
- [ ] Merchant dropdown pesquisável imediatamente (não depende de location)
- [ ] Selecionar merchant → Location é limpo e recarregado com locations válidas desse merchant
- [ ] Selecionar location (com merchant vazio) → merchant auto-preenche o correspondente
- [ ] Salvar → toast de sucesso; merchant/location refletem seleção atualizada
