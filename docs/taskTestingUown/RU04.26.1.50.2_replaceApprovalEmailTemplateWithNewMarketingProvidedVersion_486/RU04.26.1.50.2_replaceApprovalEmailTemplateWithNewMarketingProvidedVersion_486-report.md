## Achado de Infraestrutura

O `ApplicationNotificationListener` em STG **nao esta disparando emails** via o path async (`@Async("notificationExecutor")` + `@TransactionalEventListener(AFTER_COMMIT)`). Os leads sao aprovados com `UW_APPROVED` e `max_approval_amount > 0`, mas nenhum registro e criado na `uown_email_queue` pelo path automatico.

**Workaround aplicado:** Chamada explicita a `getFinalApprovalDetails(leadPk)` que usa o path sincrono (`ApprovalEmailService.sendApprovalEmail(lead)` com `async=false`).

**Causa provavel:** Thread pool `notificationExecutor` morto ou saturado (config: corePoolSize=3, maxPoolSize=10, queueCapacity=100). O listener engole erros silenciosamente (sem log quando `lead == null` ou quando o executor rejeita a task).
