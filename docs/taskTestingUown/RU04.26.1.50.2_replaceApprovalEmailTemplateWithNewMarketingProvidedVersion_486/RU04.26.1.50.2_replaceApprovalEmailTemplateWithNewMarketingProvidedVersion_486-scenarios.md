# Task #486 — Replace Approval Email Template with New Marketing-Provided Version

## Contexto

O template de e-mail de aprovacao deve ser substituido pelo template "No Money Down" V2 do marketing. Todas as imagens devem ser hospedadas no bucket GCS (storage.googleapis.com/uown/). O sistema envia templates diferentes para UOWN vs Kornerstone merchants.

**Trigger**: `sendApplication` com lead aprovado (SSN nao terminando em 9) → evento `ApplicationApprovedEvent` → `ApprovalEmailService` → `CorrespondenceService` → EmailQueue

**Template selection**: company=KORNERSTONE → `KORNERSTONE_ApprovalEmail`, senao → `ApprovalEmail`

---

## US-EMAIL-01: Validacao do E-mail de Aprovacao UOWN

**Como** QA engineer,
**Quero** validar que o e-mail de aprovacao UOWN usa o novo template com conteudo e imagens corretos,
**Para** garantir que o branding esta consistente e os assets estao hospedados no GCS.

### Criterios de Aceite
- [ ] E-mail recebido com subject contendo "Approved" e "Uown"
- [ ] Body contem nome do cliente, valor de aprovacao, nome do merchant
- [ ] Banner "NO MONEY DOWN!" presente (quando sem processing fee)
- [ ] Todas as URLs de imagem apontam para `storage.googleapis.com/uown/`
- [ ] Todas as imagens retornam HTTP 200 (sem broken images)
- [ ] Links/botoes apontam para destinos corretos (uownleasing.com)
- [ ] Footer contem endereco e telefone da Uown

---

**Tipo:** Hybrid (API + Email IMAP)
**Pré-condicao:** Merchant UOWN (TerraceFinance), ambiente sandbox

**Passos:**
1. Enviar `sendApplication` via API com SSN nao terminando em 9 (low risk tier)
