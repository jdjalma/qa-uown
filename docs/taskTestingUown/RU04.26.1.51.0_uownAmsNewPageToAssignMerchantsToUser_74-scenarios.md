## Contexto da Feature

Primeira parte de refatoração de UX no gerenciamento de usuários do AMS:
1. **User Details Page** (`/users/[username]`): os detalhes do usuário (antes exibidos como painel lateral na listagem) agora são uma página dedicada. Funcionalidade deve ser equivalente ao painel lateral anterior.
2. **User Merchant Association Page**: nova página com **duas tabelas selecionáveis com paginação** para associar merchants a usuários. A operação é **aditiva** — nunca remove associações existentes.

**Endpoint crítico:** `POST /user/addMerchantsToUsers`  
Body: `{ merchantRefCodes: string (comma-delimited), usernames: string[] }`  
Regra: se `user.merchantCodes` contém `"*"` → usuário tem acesso global, associação é **pulada**.

---

## US-AMS-01: Página de Detalhes do Usuário

**Como** administrador do AMS,  
**Quero** acessar uma página dedicada de detalhes do usuário em `/users/[username]`,  
**Para** visualizar e editar as informações do usuário com a mesma funcionalidade do antigo painel lateral.

### Critérios de Aceite
- [ ] Clicar em um usuário na lista navega para `/users/[username]`
- [ ] A página exibe os campos principais do usuário: username, email, telefone, primeiro/último nome
- [ ] É possível editar e salvar dados do usuário na página de detalhes (`PUT /user/{username}`)
- [ ] As alterações persistem após reload da página

---

## US-AMS-02: Página de Associação de Merchants a Usuário

**Como** administrador do AMS,  
**Quero** usar a nova página de associação de merchants com duas tabelas selecionáveis,  
**Para** vincular múltiplos merchants a um usuário de forma aditiva.

### Critérios de Aceite
- [ ] A página de associação é acessível a partir da página de detalhes do usuário
- [ ] Duas tabelas selecionáveis são exibidas: merchants disponíveis (não associados) e merchants já associados / selecionados
- [ ] Selecionar merchant(s) da tabela de disponíveis e confirmar associa corretamente
- [ ] A seleção na tabela reflete na outra tabela (estado visual da associação)
- [ ] Merchants novos são **adicionados** aos já existentes (não substituídos)
- [ ] Paginação funciona: navegar para página 2 e selecionar merchant de lá mantém a seleção
- [ ] Tentativa de associar merchant já associado não causa duplicata
- [ ] Usuário com `merchantCodes = "*"` não tem seus merchants alterados pela API
