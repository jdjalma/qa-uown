### Cenário: Cenário 1 — CT-UI-01

**O que é feito:** Abre a listagem de usuários e clica no primeiro usuário da tabela.

**O que acontece:** O sistema redireciona para a página de detalhes dedicada (`/users/ysantos.gow`) em vez do antigo painel lateral. O botão de edição (lápis) aparece na página.

**O que é verificado:** A URL aponta para a rota de detalhes do usuário e o botão Edit está visível, confirmando que a nova página de detalhes carrega corretamente.

Exemplos:
| Campo | Valor |
|-------|-------|
| URL navegada | `https://ams-website-qa2.uownleasing.com/users/ysantos.gow` |
| Edit button visível | `true` |

### Cenário: Cenário 7 — CT-UI-07 (Bug Fix Verification)

**O que é feito:** Captura o último registro do Log Activity do usuário, atualiza os merchants via `PUT /user/{username}` (novo mecanismo) e recarrega a página para verificar se o log foi gerado.

**O que acontece:** O PUT com `merchantCodes` no body gera uma entrada no Log Activity, ao contrário do antigo `POST /user/addMerchantsToUsers` que não logava essa operação.

**O que é verificado:**
1. As tags dos merchants aparecem corretamente no card "Edit User Merchants"
2. O Log Activity registra a alteração de merchants (antes só mostrava a alteração anterior de telefone)
3. O bug está corrigido — operações de merchants agora são logadas pelo novo endpoint

Exemplos:
| Campo | Valor |
|-------|-------|
| testUsername | `evedovatto.gow_clone` |
| merchantCodes enviados via PUT | `214,B082922-10012143307` |
| Merchant tags visíveis | `214`, `Bridge12143307 - B082922-10012143307` |
| Log Activity baseline (última entrada) | `UPDATED user info: {"phoneNumber":"5559999999"}` |
| Log Activity após PUT (última entrada) | `UPDATED user info: {"merchantCodes":"214,B082922-10012143307"}` |
| **Bug fix confirmado** | **SIM** — PUT /user/{username} gera Log Activity, ao contrário do antigo POST /user/addMerchantsToUsers |

#### Mudança de Mecanismo

| Aspecto | Antes | Agora |
|---------|-------|-------|
| Endpoint | `POST /user/addMerchantsToUsers` | `PUT /user/{username}` com `merchantCodes` |
| Gera Log Activity | **NÃO** (bug) | **SIM** (corrigido) |
| Comportamento | Aditivo (só adiciona) | Substitutivo (envia lista completa) |

**PASSOU**
