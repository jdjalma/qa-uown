## Test Strategy

- **Approach:** **Hybrid, UI-first (rule #14).** Setup via API where possible; **exercise + assert the rendered contract via the browser / PDF flatten** — content bugs (blank tokens, wrong tail, plural appendix) are only detectable in the render, never in a log. DB used for routing assertion (`uown_esign_document`), token cross-check (`request.variables`), and activity-log validation (`uown_los_lead_notes`).
- **Justification:** the feature is a customer-facing rendered legal document. Rule #14 forbids API-only here; the Daniel's Jewelers CA bug (empty placeholders in the PDF) is the origin precedent and is directly analogous to the TN `salesTax`/`nextPaymentDueAmount` blank risk.
- **Two execution tiers (dictated by the NeuroID blocker):**
  1. **Automatable (CI):** `TN_2025_SAC` (13m) via **TerraceFinance** (ONLINE, UOWN gateway `secure-stg.uownleasing.com`, no NeuroID). Covers CT-01's **negative control** (13m → `TN_2025_SAC`, GOWSIGN) and provides CT-07's **sanity reference**. Uses the existing helper chain (`createPreQualifiedApplication` → `sendInvoice` → `MissingDataFormPage`/`ContractCompletePage` → `TermsOfAgreementPage` → `AlternativeContractModalPage` → `signGowSignInFrame`).
  2. **MCP-manual only (NOT CI):** `TN_2025_SAC_16_MONTHS` (16m) via a **Kornerstone ONLINE merchant whose `valid_states` includes TN**. The Kornerstone `/complete` host carries the NeuroID gate; IL (#576) proved the API `submitApplication` returns `"Failed to verify identification."` (no `uown_esign_document`, lead stuck `UW_APPROVED`) and the browser auto-advance is blocked too. The documented workaround (reload → "Sign Contract") was proven only via Playwright MCP, and there is no page object for that surface. **Consequence:** CT-02..CT-06 (16m content) + CT-08 (16m signing) are, today, validatable only by an MCP-manual render pass — the exact wording must be frozen from that render before the oracle checkpoints promote from `[expected]` to `[confirmed]`.
- **Mandatory first step — discovery pass (rule #18, UI→API→DB):** before any implementation, confirm (a) `getGowSignTemplatesForState(db,'TN')` returns both `TN_2025_SAC` (13m) and `TN_2025_SAC_16_MONTHS` (16m) in the target env; (b) which Kornerstone ONLINE merchant lists TN in `valid_states` and returns EligibleTerms 16 (the IL analogue was KS3015 "5th Ave Furniture (NY)"); (c) whether the NeuroID gate is present on that merchant's TN route. Document findings in a new `docs/knowledge-base/tennessee-gowsign-template.md` (mirror `illinois-gowsign-template.md`). This pass also freezes the canonical render text.
- **Environments:** **stg** primary — it is where IL/AL/OH/LA reached the 16m Kornerstone route and where the templates are DB-confirmed for siblings; DoD also requires stg. **qa2** as a secondary check for the automatable 13m tier (its TN template availability is UNVERIFIED — re-list before relying on it). Avoid qa1 for `sendApplication` (DV360 outage precedent). sandbox for local iteration of the 13m automatable path.
- **Suites to activate (`regression-suites-map` / `gowsign-knowledge`):** because a template changed → `gowsign-contract-content*` + `gowsign-signature-fields` are the relevant families. SignWell visual diff is **N/A for TN** (TN routes GowSign, not a GowSign↔SignWell swap on the same state). Cross-reference the sibling task dirs (IL #576, IN #579, VA #568, MA #583, TX #515, AL #555, OH, NY) for shared page objects/helpers — do NOT recreate (rule #2).
- **Merchant preflight (rule #12):** required for the fresh-lead setup (13m TerraceFinance tier and the 16m Kornerstone tier) — the lead must reach `UW_APPROVED` with a valid 16m program on the Kornerstone merchant for the 16m tier. Do NOT run preflight against an existing lease. Snapshot immutability: the 16m program must be active BEFORE UW approval.
- **Smoke vs full:** the automatable 13m routing + sanity reference (CT-01 negative control, CT-07 reference) can be a smoke-level guard; the 16m content CTs are full/manual by nature of the blocker.

### CT-03 — Item 4a Lease-Purchase Ownership: TN sales-tax + processing-fee tail  `[P0]`
- **Technique:** Decision-table negative seeding + Error Guessing (homogenization). The interesting columns are the sibling-state patterns that MUST NOT appear.
- **Persona:** Tennessee customer signing a 16-month lease (Website/GowSign iframe); the rendered legal tail is what they read to understand total cost and ownership.
- **Setup:** fresh TN 16m lead (Kornerstone route) → CONTRACT_CREATED → GowSign render (MCP-manual per blocker).
- **Steps:**
  1. Render `TN_2025_SAC_16_MONTHS` in the GowSign iframe; extract normalized body text.
  2. Assert the MUST tail (see below).
  3. Assert every MUST-NOT pattern is absent.
- **Validations:**
  - UI/PDF MUST: "current to elect EPO"; EPO price = daily lease charges from inception through the exercise date + fees/taxes − on-time payments (excluding taxes/fees); late fees not included; no ownership until paid in full; tail `/[\d,]+ payments of \$\s?[\d.,]+ plus \$\s?[\d.,]+ in sales tax/` + "and you pay the processing fee" + "Total Cost" + `$ [contractAmount]` + "and you will own the Property".
  - UI/PDF MUST NOT: `/of \$\s?[\d.,]+ \(plus tax\)/` (the IL/VA/IN/MA `(plus tax)` pattern — the exact homogenization risk); `/\d+%\s*discount/`; "remaining lease payments for ownership" (standard-SAC discount math); `/\d+-Day-Promotional-Payoff Option \(Item 4\)/` (standard-SAC reference).
  - DB: `salesTax` and `nextPaymentDueAmount` render non-empty (cross-checked below in CT-04).
  - Activity log: covered by CT-08.
- **Edge cases covered:** wrong-donor copy (VA/IL tail), stale standard-SAC discount clause left in the 16m template.
- **Pitfalls considered:** float repr on the money tokens (`toBeCloseTo`, not `toEqual`); iframe host varies by env (use the class selector, not `gowsign.com`).
