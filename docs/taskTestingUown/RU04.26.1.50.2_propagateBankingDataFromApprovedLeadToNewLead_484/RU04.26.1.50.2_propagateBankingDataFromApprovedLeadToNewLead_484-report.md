## Descrição

Nova etapa `BankDataStep` adicionada após `underwritingCheck` no pipeline `sendApplication`. Quando um cliente retornante aplica SEM banking data, e seu lead anterior aprovado possuía banking data (source=SEND_APP) e era elegível para termos de 16 meses, o banking data é automaticamente propagado para o novo lead em `uown_los_bank_account`.

Condições para propagação (TODAS devem ser atendidas):
1. Config flag `saveBankDataFromPreviousLead = true` (default)
2. UW status = APPROVED
3. `eligibleTerms` contém "16" (merchant elegível para 16 meses)
4. `previousUW` não vazio (cliente retornante com UW anterior)
5. Request NÃO inclui banking data (`mainBankAccountNumber` ausente)
6. Lead anterior tem ao menos uma bank account com source = SEND_APP
