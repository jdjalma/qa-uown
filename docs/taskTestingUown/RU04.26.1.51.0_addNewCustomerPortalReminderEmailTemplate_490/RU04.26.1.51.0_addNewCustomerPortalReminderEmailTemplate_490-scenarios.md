## Cenarios de Teste

### CT-01: Enviar Customer Portal Reminder para conta UOWN e verificar recebimento do email

**Tipo:** API + Email (IMAP)
**Portal:** Servicing (API)
**Pre-condicao:** Conta ACTIVE com company=UOWN, email de teste configurado

**Passos:**
1. Buscar/identificar conta UOWN ACTIVE via DB
2. Alterar email da conta para email de teste via API (`createOrUpdateEmail`)
3. Chamar `POST /uown/svc/sendCustomerPortalLink/{accountPk}` para disparar o email
4. Chamar `POST /uown/svc/sendEmailsSweep` para processar a fila
5. Aguardar email via IMAP (`getEmailContent` com subject pattern)
6. Salvar HTML do email como artefato
7. Tirar screenshot do email renderizado no browser
8. [reflex] Verificar registro na uown_email_queue com template_name='CustomerPortalReminderEmail'

**Resultado esperado:**
- API retorna 200 com mensagem de sucesso
- Email recebido com template UOWN
- Template contem nome do cliente, logo UOWN, link para portal

**Tags:** @regression @critical

---

### CT-02: Validar conteudo do template UOWN (body, imagens, links)

**Tipo:** Validacao de conteudo
**Portal:** N/A (analise do HTML recebido)
**Pre-condicao:** Email UOWN recebido no CT-01

**Passos:**
1. Extrair texto do body HTML (strip tags)
2. Verificar conteudo obrigatorio: "Uown Customer Portal", "manage your account", "verification code"
3. Verificar conteudo exclusivo: NAO contem "Kornerstone"
4. Extrair URLs de imagens e verificar que apontam para GCS
5. Verificar que todas as imagens retornam HTTP 200
6. Extrair links e verificar destinos (portal, social media)

**Resultado esperado:**
- Body contem texto esperado do template UOWN
- Todas as imagens no GCS e acessiveis (200)
- Links apontam para dominio UOWN

**Tags:** @regression @critical
