## Descrição

Um novo template de email deve ser adicionado para comunicações de Customer Portal Reminder. O template é usado para lembrar clientes sobre o acesso ao Customer Portal e as ações que podem realizar por lá. O template deve ser implementado com base no HTML fornecido e deve estar disponível para ambas as marcas: UOWN e Kornerstone. Todas as imagens devem ser hospedadas no bucket GCS.

## Cenários

### CT-01

**Objetivo:** Verificar que o sistema dispara o email de Customer Portal Reminder para uma conta UOWN e que o email é recebido corretamente na caixa de entrada.

**O que é verificado:** Ao chamar a API `sendCustomerPortalLink` para uma conta ACTIVE com company=UOWN, o sistema enfileira e envia um email com o template UOWN. Após o `emailSweep`, o email é entregue ao destinatário com o assunto "Customer Portal Reminder Email" e o HTML contém o branding UOWN completo (logo, cores, portal URL). O teste também confirma que o endereço de email original da conta é restaurado após o envio.

Exemplos:
| Campo | Valor |
|-------|-------|
| accountPk | 11268 |
| customerFirstName | Mary |
| company | UOWN |
| Email de teste | fintechgroup777+490_uown_8882201_595670@gmail.com |
| Subject recebido | Customer Portal Reminder Email |
| API Response | `{"message":"Customer portal reminder email and SMS sent successfully.","errorMessage":null}` |

#### Como verificar manualmente

1. Executar `SELECT pk, account_status, company FROM uown_sv_account WHERE pk = 11268` — confirmar status=ACTIVE e company=UOWN (ou NULL para UOWN).
2. Chamar `POST /uown/svc/sendCustomerPortalLink/11268` autenticado com token Servicing — esperar HTTP 200 com body contendo `"message":"Customer portal reminder email and SMS sent successfully."`.
3. Chamar `POST /uown/svc/sendEmailsSweep` para processar a fila de emails.
4. Verificar na caixa do email de teste que o email chegou com subject "Customer Portal Reminder Email".
5. Abrir o email e confirmar branding UOWN: logo azul, menção ao nome do cliente, botão "Start your Customer Portal" apontando para `portal.uownleasing.com`.

**PASSOU**

| status | SENT |
| from_email_address | uown.dev@uownleasing.com |
| account_pk | 11268 |

#### Como verificar manualmente

1. Executar no banco de dados:
   ```sql
   SELECT eq.pk, eq.template_name, eq.status, eq.to_email_addresses, eq.from_email_address
   FROM uown_email_queue eq
   WHERE eq.account_pk = 11268
     AND eq.template_name = 'CustomerPortalReminderEmail'
   ORDER BY eq.row_created_timestamp DESC
   LIMIT 1;
   ```
2. Confirmar que `template_name = 'CustomerPortalReminderEmail'`.
3. Confirmar que `status = 'SENT'`.
4. Confirmar que `from_email_address = 'uown.dev@uownleasing.com'` (ou o remetente do ambiente atual).
