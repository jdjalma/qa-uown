## Executive summary (decisões)

- Comportamento observável = **persistência em DB** (2 colunas INTEGER gravadas no momento do underwriting), **sem UI própria**. Mesmo formato das migrations anteriores (`lambda_segment`, `is_eligible_for_extra_info`, `internal_decision`) → validação **read-only em DB após fluxo real**.
- UI-first (regra #14) satisfeito por exercitar o **gatilho** (application → send bank data → underwriting) via fluxo real; os campos novos são **validação cross-cutting de DB** (exceção c da #14). Não há superfície de render → nada visual a regredir.
- `npm_segment` e `tam_score` têm **alcançabilidade diferente → CTs separados**:
  - `npm_segment` — esperado em **qualquer** application decidida por GDS que o retorne.
  - `tam_score` — **só TireAgent** (caveat do Marcos). Colide com a restrição de 16m por ambiente → dirige a escolha de env.

## Scope

**IN:**
- `uown_los_uwdata.npm_segment` populado após application (bank data + 16m, modalidade SSN da task) atingir decisão UW **GDS** — AC core, lado lead.
- `uown_los_uwdata.tam_score` populado para application **TireAgent** nas mesmas condições — AC core, campo TireAgent-only.
- `tam_score = NULL` para application **não-TireAgent** com `npm_segment` ainda populado — partição de controle (prova que o caveat é um branch real, não flake).
- Snapshot Servicing: `uown_sv_uwdata.npm_segment`/`tam_score` após funding → account (migration tocou AS DUAS tabelas; Marcos citou só `los_`).
- Guard de atribuição de motor: `uown_los_uwdata.decided_by_agent = 'GDS'` nas linhas sob teste (campos vêm do parser GDS; ABB/Taktile deixam NULL legitimamente).
- Activity log (regra #13): a decisão UW é transição de status → confirmar nota em `uown_los_lead_notes` para o evento de decisão.

**OUT:**
- Assert visual/render dos campos (sem superfície de UI — inventar requisito).
- Lógica de scoring interna do GDS / faixas de valor (concern de vendor; QA valida persistência + não-nulo, não correção numérica).
- Matriz completa de modalidades SSN (13m-only, denied, etc.) — estes campos são específicos do caminho **16m + GDS + bank-data**. Omissão explícita.
- Linhas de revisão em `uown_los_uwdata_history`.
