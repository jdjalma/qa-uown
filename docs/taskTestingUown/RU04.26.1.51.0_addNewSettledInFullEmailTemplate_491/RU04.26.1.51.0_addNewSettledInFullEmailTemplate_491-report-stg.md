## Descrição

A tarefa adiciona o template de e-mail "Settled in Full" para ambas as marcas (UOWN e Kornerstone) e ativa o sweep agendado `settledInFullAccountEmailSweep`, que percorre contas elegíveis e dispara o e-mail de acordo com regras de negócio.

Critérios de aceite observados:

- Template "Settled in Full" existente para a marca UOWN, com imagens hospedadas no GCS (sem URLs externas).
- Template "Settled in Full" existente para a marca Kornerstone, com as mesmas regras.
- Sweep `settledInFullAccountEmailSweep` agendado via Quartz (`cronTrigger = 0 0 2 ? * MON-FRI`), ativo, usando SQL que filtra por `status = SETTLED_IN_FULL`, exclui `rating IN (E, F, U)` (ou `rating IS NULL`), respeita `do_not_email` e aplica janela temporal dependente do DOW.
- E-mail disparado apenas para contas com status `SETTLED_IN_FULL`, rating elegível (≠ E/F/U) e `do_not_email = false`.

## Bugs de Aplicação Encontrados

### TPL-01 — `SettledInFullEmail` (UOWN) referencia URL `https?://(www\.)?uownleasing\.com` no corpo

**Severidade:** Média (depende de interpretação) — **Possível bug de template OU asserção overconstrained do teste**.

**Sintoma:**
- E-mail UOWN entregue via IMAP contém pelo menos uma URL com domínio `uownleasing.com` (regex `/https?:\/\/(www\.)?uownleasing\.com/i` matchou no body lowercased).
- A asserção em `spec.ts:472` proíbe explicitamente esse padrão.
- O critério de aceite explícito da task fala em "imagens hospedadas no GCS (sem URLs externas)" — não em "nenhum link `uownleasing.com` em qualquer lugar do e-mail".

**Hipóteses:**
1. **Bug do template:** o template embute um link comercial/legal (footer/CTA) apontando para `https://www.uownleasing.com` quando deveria estar usando um link relativo, GCS-hosted, ou removido.
2. **Asserção overconstrained:** a regra de teste extrapolou o critério. Caso o link no rodapé seja legítimo (footer com endereço da empresa), o teste deveria permitir `www.uownleasing.com` e proibir apenas hosts não-GCS para imagens (já coberto por CT-03).

**Ação sugerida:**
- Inspecionar o template `SettledInFullEmail` para localizar a URL ofensora.
- Decisão de produto: o link público pertence ao template ou deveria ser removido?
- Se manter: relaxar a asserção em `spec.ts:472` para excluir o link específico (ex.: permitir `https://www.uownleasing.com` raiz e proibir apenas `cdn.uownleasing.com`, `tracking.uownleasing.com`, etc.).
- Se remover: dev fix no template e re-rodar.

**Evidência:**
- `reports/491-execution-run31-stg.log` linhas do step `verify: no prod uownleasing domain leak`.
- E-mail enviado: `sentTime=2026-04-27T12:15:15.846Z`, accountPk=589066.
