# RU03.26.1.50.0 · Query-13 · getLosSimpleSearchResults (#463)

**GitLab:** https://gitlab.com/uown/backend/svc/-/work_items/463
**Milestone:** Uown | RU03.26.1.50.0

## What was optimized

The `getLosSimpleSearchResults` query is the quick lead search by name in the Origination and Servicing portals. It was causing high CPU usage because it compared names case-insensitively (`UPPER(first_name) = UPPER(:param)`) without an index, forcing a full table scan on every search.

The optimization added 2 functional indexes that store the `first_name` and `last_name` values already uppercased. With this, the database locates the customer directly via the index without scanning the entire table, regardless of how the agent typed the name in the search box.

### CT-02 — The functional indexes contain the correct UPPER() expression

Verifies that each index was created with the `UPPER()` expression over the correct column, by querying the DDL definition in `pg_indexes`. An index created without the functional expression would not be used by the query execution plan.

| Index | DDL Definition | Result |
