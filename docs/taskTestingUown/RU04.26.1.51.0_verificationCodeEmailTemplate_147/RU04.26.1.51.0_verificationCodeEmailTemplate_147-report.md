3. Consultar a tabela `uown_email_queue` no banco qa2:
   ```sql

   FROM uown_email_queue
   ```

## Bugs de Aplicação Encontrados

### BUG-01 — Novo template de email de verificação não implantado no qa2

**Status:** OPEN
**Severidade:** High

**Descrição:** O novo template profissional de email de verificação do MR `svc!1356` (merged em 2026-04-10) NÃO foi implantado no ambiente qa2. O corpo do email na tabela `uown_email_queue` contém apenas texto simples: `"Your verification code is 665988"` (32 caracteres). O template esperado é um HTML profissional com aproximadamente 6000+ caracteres, contendo logo da empresa, ícone de cadeado, código em caixas estilizadas, informações de contato e links de redes sociais.

**Triagem (Bug Triage Protocol):**
- **Passo 1 — Comportamento errado?** SIM. A tarefa exige explicitamente o novo template. O email enviado usa o formato antigo (texto simples), confirmado via consulta à tabela `uown_email_queue`.
- **Passo 2 — Em escopo?** SIM. A tarefa inteira (#147) trata da criação e implantação do novo template. CT-01 valida exatamente este requisito.
- **Passo 3 — Regressão?** Não é regressão — é uma lacuna de deploy. O código foi merged no branch R1.51.0 mas o SVC não foi atualizado no qa2.
- **Conclusão:** Bug em escopo, confirmado. Reportado como BUG-01.

**Como Reproduzir:**
1. Acessar o Customer Portal em `https://website-qa2.uownleasing.com/`
2. Inserir email `fintechgroup777@gmail.com` e submeter
3. Consultar a tabela `uown_email_queue`:
   ```sql
   SELECT pk, subject, email_body, template_name, LENGTH(email_body) as body_length
   FROM uown_email_queue
   WHERE LOWER(to_email_addresses) LIKE '%fintechgroup777@gmail.com%'
     AND LOWER(template_name) LIKE '%verification%'
   ORDER BY pk DESC
   LIMIT 1;
   ```
4. Observar: `email_body` = `"Your verification code is XXXXXX"` (texto simples, ~32 caracteres)
5. Esperado: HTML completo com `<!DOCTYPE`, `lock_icon`, logo, contato, redes sociais (~6000+ caracteres)

**Evidência:** Consulta ao banco retorna `email_body` com 32 caracteres em texto simples. Ausência de `<!DOCTYPE`, `lock_icon`, ou qualquer marcação HTML confirma que o template antigo está ativo.

**Cenário que detectou:** CT-01

**Causa provável:** O MR `svc!1356` foi merged no branch R1.51.0 em 2026-04-10, mas o serviço SVC não foi redeployado no ambiente qa2 com esta versão. O template Thymeleaf `verification-code-email.html` existe no código-fonte mas não está sendo utilizado pelo ambiente.
