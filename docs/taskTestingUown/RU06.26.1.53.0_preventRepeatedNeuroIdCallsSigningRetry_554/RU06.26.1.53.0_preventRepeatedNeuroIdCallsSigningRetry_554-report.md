## Descrição

Quando o cliente retorna ao portal de signing após um primeiro submit (CC ou bank já salvos),
um retry/continue NÃO deve disparar uma NOVA chamada de verificação NeuroID no backend. A primeira
tentativa chama NeuroID; retornos subsequentes reutilizam a verificação anterior. Um resultado
`NOT_ENOUGH_INTERACTION_DATA` não pode negar o cliente (AC-03). A primeira tentativa legítima deve
continuar chamando NeuroID (AC-04, regression guard).

## Observações

### O-01 — [OBSERVAÇÃO] env-blocker: `sendApplication` 500 (nested 401) em qa2
- **Tipo:** Observação (env-blocker, família pitfall #39 de [[application-lifecycle]])
- **Severidade:** S2 (bloqueia toda a suíte) / **Prioridade:** P1 (resolver antes de re-validar)
- **Descrição:** `/uown/los/sendApplication` retorna 500 envolvendo `HttpClientErrorException$Unauthorized: 401` — chamada interna backend-para-backend do qa2 falhando por auth. Nossa auth de edge está OK (recebemos JSON Spring estruturado, não 401 de edge).
- **NÃO é:** bug da feature svc#554, nem bug do teste. `[api-response:/uown/los/sendApplication,500]`
- **Ação:** re-executar após o time de plataforma restaurar o serviço downstream em qa2 (token de serviço). Sem fix de código aplicável.

### O-02 — [OBSERVAÇÃO] CT-02: `NOT_ENOUGH_INTERACTION_DATA` não reproduzível em qa2 (N/A por design)
- O discovery probe (2026-06-15) só observou `neuro_id_status ∈ {SUCCESS, PROFILE_NOT_FOUND}` em qa2. O enum carrega `@unconfirmed`. CT-02 trata não-match como N/A (não falha a suíte), confirmando apenas que o cliente não é negado. AC-03 permanece **parcialmente coberto por design** até captura de uma row real com esse status. `[svc-source:enums.ts:36-45]`

### O-03 — [OBSERVAÇÃO] Seleção de projeto: spec fora de `docs/taskTestingUown/`
- O comando pedido (`--project=qa2`) não existe: projetos Playwright são tag/portal-based + `ENV` seleciona ambiente. A spec vive em `tests/e2e/servicing/` e foi executada pelo projeto **`servicing-ui`** (testDir `./tests/e2e/servicing`).
- O projeto `task-testing-servicing` tem `testDir: ./docs/taskTestingUown` + `grep: /@servicing/`; **não** pegaria esta spec por estar fora desse diretório. Para alinhar com a convenção de task-tests (testing.md § Naming), considerar mover a spec para `docs/taskTestingUown/{testName}/` OU manter em `tests/e2e/servicing/` cientemente. Não bloqueia validação — apenas afeta qual projeto/baseURL/storageState é usado. `[doc:playwright.config.ts:241-262]`

### O-04 — [OBSERVAÇÃO] DB probe direto indisponível nesta sessão
- O `UOWN_DB_URL_QA2` é JDBC/tunnel (`jdbc:postgresql://127.0.0.1:5445/svc`); o helper do framework parseia, mas probe `pg` standalone falhou (`ENOTFOUND` — tunnel não ativo).
