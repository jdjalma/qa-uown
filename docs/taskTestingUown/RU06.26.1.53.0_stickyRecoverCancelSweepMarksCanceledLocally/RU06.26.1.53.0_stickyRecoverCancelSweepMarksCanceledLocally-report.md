## Descrição

Regressão de backend. O `StickyRecoverCancelSweep` é o sweep que cancela uma recovery Sticky in-flight quando a conta SVC associada **deixa de estar `ACTIVE`** (PAID_OUT / CANCELLED / CHARGED_OFF / SOLD) — não faz sentido perseguir cobrança numa conta fechada.

**Bug que o fix corrige:** quando o Sticky **rejeita** o cancel com `HTTP 400 "Cannot cancel transaction"` (a transação já é terminal do lado deles), o código antigo lançava `IllegalStateException`, deixava `recovery_status` não-terminal, e o sweep **re-selecionava a mesma linha a cada execução** → falhas repetidas + emails `[UOWN] Errors in StickyRecoverCancelSweep`. O fix detecta essa rejeição específica e **marca a recovery `CANCELED` localmente**, tratando a linha como processada com sucesso (sem throw, sem falha de sweep). Ator: scheduled task (Quartz), `created_by=SYSTEM`. **Sem afordância de UI** (sweep interno — regra #18 exceção a).
