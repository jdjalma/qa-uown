## Contexto

Na release anterior, quando o EPO de contratos de 16 meses resultava negativo (cliente pagou mais que o buyout), o sistema retornava o `contractBalance`. Após discussão, decidiu-se retornar **0** em vez de valor negativo.

**Método alterado:** `PayOffAmountService.getAnytimeBuyout()` — quando `kwBuyoutAmount <= 0` ou `daysUsed <= 0`, agora retorna `BigDecimal.ZERO`.

## User Stories

### US-EPO-01: EPO retorna zero quando cliente pagou a mais (16 meses)

**Como** agente de servicing,
**Quero** que o EPO de contas de 16 meses exiba 0 quando o cliente pagou a mais,
**Para** não confundir o cliente com valores negativos.

#### Critérios de Aceite
- [ ] `GET /uown/svc/getPayoffAmount/{accountPk}` retorna `0` (não negativo) para conta 16m com overpayment
- [ ] `GET /uown/svc/getAccountSummary/{accountPk}` retorna `epoBalance: 0` para conta 16m com overpayment
- [ ] `GET /uown/svc/getServicingInfo/{accountPk}` retorna `epoBreakdown` com cálculo detalhado
- [ ] Contas de 13 meses não são afetadas pela mudança

## Queries de Suporte (DB)

```sql
-- Contas de 16 meses ativas
SELECT a.pk, a.status, ss.term_in_months, ai.activation_date,
       ss.money_factor, ss.cost_without_tax_and_fees
FROM uown_sv_account a
JOIN uown_sv_sched_summary ss ON ss.account_pk = a.pk
JOIN uown_sv_account_info ai ON ai.account_pk = a.pk
WHERE ss.term_in_months = 16
  AND a.status NOT IN ('CANCELLED', 'SETTLED_IN_FULL', 'CHARGED_OFF')
ORDER BY a.pk;
```
