## Descrição

A query `getDataMismatchForLead` verifica se um novo candidato já possui um lead aprovado com dados divergentes (endereço, e-mail, telefone). Ela causava alto consumo de CPU por realizar **full table scan** em tabelas com milhões de registros a cada submissão de aplicação.

A otimização adicionou **4 índices** que permitem ao PostgreSQL localizar registros relevantes sem varredura completa, reduzindo o tempo de resposta em até **969×**.

Os 4 índices são compartilhados com a Task #461 (`getLeadSummaryResults`). Flyway migration: `V20260220064821`.

### Cenário: Cenário 4 — CT-04

**O que é feito:** Executa `EXPLAIN ANALYZE` em 4 queries representativas que simulam os filtros usados pela `getDataMismatchForLead`, cada uma isolando um dos índices criados:
1. `SELECT 1 FROM uown_los_uwdata WHERE uw_status = 'UW_APPROVED' AND approval_expiration_date > NOW() LIMIT 1`
2. `SELECT 1 FROM uown_los_lead WHERE row_created_timestamp > NOW() - INTERVAL '90 days' AND lead_status = 'UW_APPROVED' LIMIT 1`
3. `SELECT 1 FROM uown_los_address WHERE zip_code = '10001' LIMIT 1`
4. `SELECT 1 FROM uown_los_address WHERE state = 'NY' LIMIT 1`

Antes de cada `EXPLAIN ANALYZE`, verifica a estimativa de linhas da tabela via `pg_class.reltuples` para garantir que o planner tem dados suficientes para preferir index scan.

**O que acontece:** O PostgreSQL utiliza **Index Only Scan** em todas as 4 queries, confirmando que os índices são efetivamente usados pelo planner em tabelas com milhões de registros. Nenhuma query realiza full table scan (`Seq Scan`).

**O que é verificado:**
- `uown_los_uwdata` (~4.8M linhas): plano contém `Index Only Scan` usando `idx_uwdata_status_expiration` → tempo = 2.972 ms
- `uown_los_lead` (~6.5M linhas): plano contém `Index Only Scan` usando `idx_lead_timestamp_status` → tempo = 0.122 ms
- `uown_los_address` (~6.2M linhas): plano contém `Index Only Scan` usando `idx_address_zip_code` → tempo = 1.068 ms
- `uown_los_address` (~6.2M linhas): plano contém `Index Only Scan` usando `idx_address_state` → tempo = 1.919 ms
- Regex `/Index Only Scan|Index Scan|Bitmap Index Scan/i` aplicado ao plano retorna `true` para todos
- Regex `/Seq Scan on {tabela}/i` retorna `false` para todos

| Filtro | Tabela (linhas estimadas) | Tipo de Varredura | Tempo de Execução |
|--------|--------------------------|:-----------------:|:-----------------:|
| `uw_status` + `approval_expiration_date` | `uown_los_uwdata` (~4.8M) | Index Only Scan | 2.972 ms |
| `row_created_timestamp` + `lead_status` | `uown_los_lead` (~6.5M) | Index Only Scan | 0.122 ms |
| `zip_code` | `uown_los_address` (~6.2M) | Index Only Scan | 1.068 ms |
| `state` | `uown_los_address` (~6.2M) | Index Only Scan | 1.919 ms |
