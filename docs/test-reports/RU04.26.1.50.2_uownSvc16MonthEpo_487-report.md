## Descrição

No release anterior, foi implantada alteração para exibir o saldo contratual (`contractBalance`) quando o EPO (Early Pay-Off) fosse negativo. Após discussões internas, decidiu-se exibir **0** em vez de valor negativo nos casos em que o cliente pagou a mais (overpayment).

**Fix (commit f5f0d3af):** Em `PayOffAmountService.getAnytimeBuyout()`, quando `kwBuyoutAmount <= 0` ou `daysUsed <= 0`, retorna `BigDecimal.ZERO` em vez de buscar `contractBalance`.

Lógica de roteamento em `PayOffAmountService.getPayOffAmount()`:
- Se `termInMonths == 16` → `getAnytimeBuyout()` (onde o fix foi aplicado)
- Senão se KORNERSTONE → `getKornerstoneEpo()`
- Senão → `getEpoCalculation()`

### CT-03

**Objetivo:** Verificar que quando `daysUsed <= 0` (conta recém-ativada com activation_date = hoje), o EPO retorna 0.

**O que é verificado:** Para uma conta de 16 meses cuja data de ativação é o dia atual (daysUsed = 0), o sistema retorna EPO = 0. Este é um edge case coberto pelo mesmo fix: a condição `daysUsed <= 0` no `getAnytimeBuyout()` agora retorna `BigDecimal.ZERO`.
