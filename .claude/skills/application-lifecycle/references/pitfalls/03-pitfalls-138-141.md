| 143 | **`pre-write-validate.sh` Rule 1 bloqueia componentes standalone legitimos em `src/pages/`** — qualquer arquivo em `src/pages/**` com `export class X` sem `extends` e rejeitado pelo hook (`exit 2`). **Sintoma:** hook bloqueia escrita de `*.component.ts` / `*.controls.ts` que SAO standalone por design (nao sao page objects, sao compostos dentro de paginas). **Fix atual (workaround):** usar `class X { ... }; export { X }` (declaracao sem `export` keyword → hook nao captura). **Fix ideal (pendente):** whitelist no hook: `if [[ "$FILE_PATH" =~ \.(component|controls)\.ts$ ]]; then skip Rule 1`. Candidato a melhoria do script. Ver `src/pages/components/seon-widget.component.ts` como exemplo. Analogamente afeta `src/pages/origination/filtered-csv-download.controls.ts` (que ja usa o workaround). |
